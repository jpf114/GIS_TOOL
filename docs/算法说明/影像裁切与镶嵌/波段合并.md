# 影像裁切与镶嵌 / 波段合并

## 功能说明

将多个单波段栅格按顺序合并成一个多波段栅格。

## 输入与输出

- 输入：多个单波段栅格
- 输出：多波段栅格，建议 `.tif`

## 参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| 输入文件 | 文件路径 | 否 | 无 | 可传逗号分隔文件列表。 |
| 波段列表 | 字符串 | 否 | 无 | 额外文件列表，逗号分隔。 |
| 输出文件 | 文件路径 | 是 | 无 | 输出多波段栅格。 |

## 算法内容

波段合并用于把多个单波段栅格按给定顺序组合成一个多波段栅格。当前实现会先把所有输入文件组织成一组波段来源，再通过 VRT 的 `-separate` 模式把它们拼成虚拟多波段，再转换成实际输出文件。

## 处理流程

1. 读取 `输入文件` 和 `波段列表`。
2. 将两处输入统一拆分并拼接成最终波段文件列表。
3. 检查至少存在一个可用输入文件。
4. 逐个打开这些单波段栅格。
5. 以 `-separate` 方式构建一个多波段 VRT。
6. 将该 VRT 再转换为实际多波段输出栅格。
7. 按输入顺序保持输出波段顺序。

## 填写示例

### 方式一：全部写在 `输入文件`

```text
D:/data/b1.tif,D:/data/b2.tif,D:/data/b3.tif
```

### 方式二：`输入文件` + `波段列表`

输入文件：

```text
D:/data/b1.tif
```

波段列表：

```text
D:/data/b2.tif,D:/data/b3.tif
```

### 输出说明

- 输出结果为栅格文件。
- 建议优先使用 `.tif` 保存，便于后续在 CLI、GUI 或 GIS 软件中继续处理。

## 推荐输出文件名

```text
D:/data/merged_bands.tif
```

## 处理逻辑

- `输入文件` 和 `波段列表` 至少填一个。
- 两者会被拼接成统一输入列表，再按顺序合并。

## 结果解释

- 输出结果是一幅新的多波段栅格。
- 输出第 1、2、3... 波段顺序，直接对应输入文件列表顺序。

## 适用场景

- 单波段数据重新组装为多波段影像。
- 为后续指数计算、分类或渲染准备标准多波段输入。

## 注意事项

- 输出建议使用 `.tif`。
