# 影像裁切与镶嵌 / 裁切

## 功能说明

按矩形范围或裁切矢量对输入栅格进行裁切。

## 输入与输出

- 输入：
  - 栅格：建议 `.tif`
  - 裁切矢量：常用 `.shp` `.gpkg` `.geojson`
- 输出：裁切后的栅格，建议 `.tif`

## 参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| 输入文件 | 文件路径 | 是 | 无 | 输入栅格路径。 |
| 输出文件 | 文件路径 | 是 | 无 | 输出裁切结果。 |
| 空间范围 | 范围 | 否 | `0,0,0,0` | 格式 `xmin,ymin,xmax,ymax`。 |
| 裁切矢量 | 文件路径 | 否 | 无 | 用于裁切的矢量边界。 |

## 算法内容

裁切用于从输入栅格中截取局部区域。当前实现有两条真实处理路径：如果提供了裁切矢量，则按矢量边界裁切；如果没有裁切矢量但提供了范围，则按矩形窗口裁切。两者最终都会生成新的栅格输出。

## 处理流程

1. 读取输入栅格、输出路径、空间范围和可选裁切矢量。
2. 检查 `空间范围` 和 `裁切矢量` 至少填写一个。
3. 如果提供了裁切矢量：
4. 组装 `cutline` 和 `crop_to_cutline` 参数。
5. 调用 GDALWarp 按矢量边界裁切输出。
6. 如果未提供裁切矢量但提供了范围：
7. 将 `xmin,ymin,xmax,ymax` 转成 `projwin` 参数顺序。
8. 调用 GDALTranslate 按矩形窗口裁切输出。
9. 生成新的裁切结果栅格。

## 处理逻辑

- `空间范围` 和 `裁切矢量` 至少要填一个。
- 如果提供 `裁切矢量`，优先走按矢量裁切。
- 如果只提供 `空间范围`，走矩形窗口裁切。

## 结果解释

- 矩形裁切得到的是按给定范围截取的规则矩形栅格。
- 矢量裁切会以矢量边界为准生成结果，更适合行政区、样区或专题范围提取。

## 适用场景

- 影像局部区域提取、按行政边界裁切和分析范围准备。

## 填写示例

- 空间范围：`500000,3200000,510000,3210000`
- 裁切矢量：`D:/data/clip_boundary.gpkg`
- 推荐输出文件：`D:/out/clipped.tif`

## 输出说明

- 输出结果为裁切后的栅格文件。
- 建议优先使用 `.tif`，便于继续做栅格分析或在 GIS 软件中查看。

## 推荐输出文件名

- `裁切_result.tif`

## 注意事项

- 输出一般建议使用 `.tif`。
- `空间范围` 默认全零时会被视为未填写。
