# 影像裁切与镶嵌 / 镶嵌

## 功能说明

将多幅栅格影像拼接为一幅输出栅格。

## 输入与输出

- 输入：多个栅格文件，`输入文件` 使用英文逗号分隔
- 输出：拼接后的栅格，建议 `.tif`

## 参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| 输入文件 | 文件路径 | 是 | 无 | 多个输入路径用逗号分隔，至少 2 个。 |
| 输出文件 | 文件路径 | 是 | 无 | 镶嵌结果路径。 |
| 目标坐标系 | 坐标系 | 否 | 无 | 可选，统一输出坐标系。 |
| 重采样方式 | 枚举 | 否 | `nearest` | 可选 `nearest` `bilinear` `cubic` `cubicspline` `lanczos` `average`。 |

## 算法内容

镶嵌的目标是把多幅输入栅格统一组织成一幅输出栅格，便于后续浏览、分析和制图。当前实现并不是简单按文件顺序拼接像素块，而是走 GDAL 的 Warp 流程，在统一空间参考与重采样规则下完成多输入数据合成。

这个过程可以理解为：先把多幅输入影像放到同一个输出空间框架里，再按指定的重采样方式把像元映射到目标栅格中。因此它既可以处理原本就能直接拼接的相邻影像，也可以在需要时顺带完成坐标系统一。

## 处理流程

1. 读取 `输入文件` 参数，并按英文逗号拆分为多个输入路径。
2. 检查输入数量，少于 2 个时直接报错。
3. 逐个打开输入栅格，确认它们都能参与后续处理。
4. 构造统一的镶嵌参数，包括输出路径、重采样方式，以及可选的目标坐标系。
5. 调用 GDAL Warp 流程，将所有输入栅格投放到统一输出网格中。
6. 在重叠区域内，由 Warp 按当前流程生成最终输出结果。
7. 将镶嵌结果写出为单个栅格文件。

## 枚举值说明

### 重采样方式

- `nearest`：最近邻，速度快，适合分类值
- `bilinear`：双线性，适合连续影像
- `cubic`：三次卷积
- `cubicspline`：三次样条
- `lanczos`：高质量重采样
- `average`：均值

## 填写示例

### 多文件输入

```text
D:/data/a.tif,D:/data/b.tif,D:/data/c.tif
```

### 输出说明

- 输出结果为栅格文件。
- 建议优先使用 `.tif` 保存，便于后续在 CLI、GUI 或 GIS 软件中继续处理。

## 结果解释

- 输出结果是一幅新的完整栅格，不再保留多个源文件的分离状态。
- 如果设置了目标坐标系，输出结果会统一到该坐标系下。
- 重采样方式会影响连续影像的平滑程度和分类影像的类别保持方式，因此要按数据类型选择。

## 适用场景

- 多景遥感影像合并成单幅成果图。
- 分幅栅格整理、统一坐标系输出。
- 为后续裁切、统计、显示和发布提供连续底图。

## 推荐输出文件名

```text
D:/data/scene_mosaic.tif
```

## 注意事项

- 输入少于 2 个文件会直接失败。
- 代码按 GDAL Warp 流程执行，输出推荐使用 `.tif`。
