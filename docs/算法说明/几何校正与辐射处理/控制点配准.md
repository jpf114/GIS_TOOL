# 几何校正与辐射处理 / 控制点配准

## 功能说明

根据控制点 CSV 对输入影像执行轻量几何配准，当前基于 GDAL 一阶变换完成。

## 输入与输出

- 输入：
  - 输入影像：建议 `.tif`
  - 控制点 CSV：表头需包含 `pixel_x,pixel_y,map_x,map_y`
- 输出：
  - 配准结果栅格：建议 `.tif`

## 控制点 CSV 示例

```csv
pixel_x,pixel_y,map_x,map_y
0,0,120,30
9,0,129,30
0,5,120,25
9,5,129,25
```

## 参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| 输入文件 | 文件路径 | 是 | 无 | 输入影像 |
| 输出文件 | 文件路径 | 是 | 无 | 输出配准结果 |
| gcp_file | 文件路径 | 是 | 无 | 控制点 CSV |
| dst_srs | 字符串 | 是 | 无 | 目标坐标系，如 `EPSG:4326` |
| resample | 枚举 | 否 | `nearest` | 重采样方式 |

## 算法内容

控制点配准用于根据一组已知的像素坐标与地图坐标对应关系，对原始影像建立空间参考并输出配准结果。当前实现采用的是基于控制点的一阶几何变换流程，重点是把输入影像从“只有像素位置”或“空间参考不可靠”的状态，转换成带有目标空间参考的可用栅格。

这里的控制点文件是一个表格化输入，描述的是“图像上这个像素位置，对应地面上的哪个坐标位置”。算法不会自动识别控制点，也不会执行高阶畸变建模，而是按给定控制点直接完成基础配准。

## 处理流程

1. 读取控制点 CSV，并检查表头是否包含 `pixel_x`、`pixel_y`、`map_x`、`map_y` 这几个必需字段。
2. 逐行解析控制点记录；如果提供了 `map_z`，则一并读取。
3. 检查控制点数量，少于 3 个时无法完成当前配准流程。
4. 打开输入影像，并创建一个临时数据集副本用于挂接控制点信息。
5. 将解析出的控制点写入临时数据集，同时设置目标坐标系。
6. 以一阶变换方式执行重投影与重采样，生成新的空间配准结果。
7. 将结果写出到目标栅格文件，并保留目标空间参考信息。

## 输出说明

- 输出结果带有目标投影和变换后的空间参考
- 当前实现适合基础配准和联调验证

## 结果解释

- 输出结果的像元内容仍然来自原始影像，但空间位置已经根据控制点关系重新定位。
- 如果控制点分布合理、数量充足且误差较小，输出影像会与目标坐标系下的其他数据更容易叠加。
- 如果控制点集中在局部、存在明显误差或目标坐标系设置不当，结果可能出现整体偏移、局部拉伸或边缘误差。

## 适用场景

- 原始影像缺少可靠空间参考，需要快速建立基础地理定位。
- 已知少量控制点，需要进行轻量配准或联调验证。
- 作为更高精度几何校正前的基础处理步骤。

## 推荐输出文件名

- `gcp_register_result.tif`

## 注意事项

- 控制点数量至少需要 3 个
- 当前使用一阶配准，不包含高阶模型
