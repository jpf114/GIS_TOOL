# 几何校正与辐射处理 / 辐射定标

## 功能说明

按给定 `gain/offset` 或元数据文件中的系数，对单波段栅格执行线性辐射定标。

## 输入与输出

- 输入：单波段或多波段栅格
- 输出：单波段浮点栅格，建议 `.tif`

## 参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| 输入文件 | 文件路径 | 是 | 无 | 输入影像 |
| 输出文件 | 文件路径 | 是 | 无 | 输出定标结果 |
| 波段号 | 整数 | 否 | `1` | 参与处理的波段 |
| gain | 浮点数 | 否 | `1.0` | 乘法系数 |
| offset | 浮点数 | 否 | `0.0` | 加法系数 |
| metadata_file | 文件路径 | 否 | 无 | 系数元数据文件，存在时优先使用 |

## 算法内容

辐射定标用于把原始像元值按线性关系转换成更适合后续分析的辐射量或反射率近似值。当前实现采用最直接的线性定标方式，即对指定波段执行 `输出 = 输入 * gain + offset`。如果提供了元数据文件，则优先从元数据中读取当前波段对应的定标系数。

这意味着当前流程不做复杂的大气或传感器模型解算，而是把重点放在“按当前波段的线性系数完成转换”。

## 处理流程

1. 读取输入参数，确定处理波段、手工 `gain/offset` 和可选元数据文件。
2. 如果提供了 `metadata_file`，优先按当前波段从元数据中解析增益和偏移系数。
3. 打开输入栅格并读取指定波段。
4. 对该波段逐像元执行线性变换：`输入值 * gain + offset`。
5. 将结果转换为浮点格式。
6. 输出为单波段浮点栅格，并记录本次实际使用的系数。

## 支持的元数据键

- `RADIANCE_MULT_BAND_n`
- `RADIANCE_ADD_BAND_n`
- `REFLECTANCE_MULT_BAND_n`
- `REFLECTANCE_ADD_BAND_n`
- `BAND_n_GAIN`
- `BAND_n_OFFSET`
- `GAIN`
- `OFFSET`

## 输出说明

- 输出结果为浮点定标结果
- 使用元数据文件时，会按当前波段自动解析对应系数

## 结果解释

- 输出值是对原始像元值做线性变换后的结果。
- 如果元数据系数是辐亮度口径，输出更接近辐亮度；如果是反射率口径，输出则更接近反射率近似值。
- 结果物理含义取决于系数来源，因此后续使用时要明确当前用的是哪类定标参数。

## 适用场景

- 原始影像预处理。
- 为指数计算、校正处理或跨景比较提供更一致的输入。

## 推荐输出文件名

- `radiometric_calibration_result.tif`

## 注意事项

- `metadata_file` 存在时，会覆盖手工输入的 `gain/offset`
