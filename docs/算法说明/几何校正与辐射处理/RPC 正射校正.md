# 几何校正与辐射处理 / RPC 正射校正

## 功能说明

基于影像自带 RPC 元数据执行轻量正射校正，可选 DEM 参与。

## 输入与输出

- 输入：
  - 带 RPC 元数据的栅格影像
  - 可选 DEM
- 输出：
  - 正射校正结果栅格，建议 `.tif`

## 参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| 输入文件 | 文件路径 | 是 | 无 | 输入影像 |
| 输出文件 | 文件路径 | 是 | 无 | 输出结果 |
| dst_srs | 字符串 | 否 | `EPSG:4326` | 目标坐标系，如 `EPSG:4326` |
| dem_file | 文件路径 | 否 | 无 | 可选 DEM 文件 |
| rpc_height | 浮点数 | 否 | `0.0` | 不使用 DEM 时的固定高程 |
| resample | 枚举 | 否 | `nearest` | 重采样方式 |

## 算法内容

RPC 正射校正用于利用影像自带的 RPC 元数据，把原始影像从成像几何中纠正到目标地图坐标系下。当前实现基于 GDAL 的 `-rpc` 流程执行，重点是利用 RPC 模型建立像素与地面坐标之间的关系，并在输出时完成重投影和重采样。

如果提供了 DEM，流程会结合地形高程参与解算，结果通常更符合真实地表位置；如果没有 DEM，则使用固定高程 `rpc_height` 近似参与计算，适合轻量预处理或快速联调，但精度会受地形起伏影响。

## 处理流程

1. 打开输入影像，并检查其是否包含 RPC 元数据。
2. 确定目标坐标系和重采样方式。
3. 如果提供了 DEM，则把 DEM 作为 RPC 解算的地形输入；否则使用固定高程 `rpc_height`。
4. 构造 RPC 正射校正参数，显式启用 RPC 模型流程。
5. 调用 GDAL Warp 执行正射校正、重投影与重采样。
6. 将生成结果写出为新的栅格文件，并记录本次使用的关键参数。

## 输出说明

- 输出结果带目标投影
- 不提供 DEM 时，使用固定高程参与 RPC 解算

## 结果解释

- 输出结果仍然保留原始影像的光谱或灰度信息，但像元空间位置已经按 RPC 模型重新校正。
- 使用 DEM 时，山区或地形起伏明显区域通常会获得更合理的位置结果。
- 不使用 DEM 时，平坦地区通常仍可用于基础应用，但高差明显区域可能存在定位偏差。

## 适用场景

- 卫星影像带有 RPC 元数据，需要快速生成基础正射成果。
- 作为后续拼接、裁切、配准和专题分析前的预处理步骤。
- 有 DEM 时做更可靠的地形参与校正；无 DEM 时做轻量快速校正。

## 推荐输出文件名

- `rpc_orthorectify_result.tif`

## 注意事项

- 输入影像必须包含 `RPC` 元数据
- `dst_srs` 留空时，当前实现会按 `EPSG:4326` 处理
- 当前实现基于 GDAL `-rpc` 流程，适合轻量正射预处理
