# 栅格工具 / COG 生成

## 功能说明

将输入栅格转换为 Cloud Optimized GeoTIFF，便于云端分块读取与快速浏览。

## 输入与输出
- 输入：栅格影像
- 输出：COG 栅格，建议 `.tif`

## 参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| 输入文件 | 文件路径 | 是 | 无 | 输入栅格影像 |
| 输出文件 | 文件路径 | 是 | 无 | 输出 COG 文件 |

## 算法内容

COG 生成用于把普通栅格转换为 Cloud Optimized GeoTIFF。当前实现基于 GDALTranslate，以 `COG` 作为输出格式，并固定启用 `LZW` 压缩和 `BIGTIFF=IF_SAFER` 选项，生成更适合范围读取和云端访问的 GeoTIFF。

## 处理流程

1. 打开输入栅格。
2. 组装 COG 输出参数，固定使用 `COG` 格式、`LZW` 压缩和按需 BigTIFF。
3. 调用 GDALTranslate 执行格式转换。
4. 生成新的 COG 输出文件，并在元数据中记录格式和压缩方式。

## 输出说明

- 输出结果为 GeoTIFF
- 适合后续用于 Web 分发、云存储或高效浏览

## 结果解释

- 输出文件仍然是 GeoTIFF，但内部组织方式更适合分块读取。
- 转换重点是优化访问结构，不会主动修复源数据中的质量问题。

## 适用场景

- 云端发布、Web 浏览和大影像高效读取准备。

## 推荐输出文件名
- `output_cog.tif`

## 注意事项

- 输出文件通常比原始普通 TIFF 更适合范围读取
- 如果输入文件本身质量较差，转换不会自动修复数据问题
