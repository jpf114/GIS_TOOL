# 栅格工具 / NoData 设置

## 功能说明

为单个波段或全部波段写入 NoData 值。

## 输入与输出

- 输入：栅格文件
- 输出：原地修改输入文件，不生成新文件

## 参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| 输入文件 | 文件路径 | 是 | 无 | 输入栅格。 |
| 波段号 | 整数 | 否 | `1` | 代码中 `<= 0` 表示对全部波段生效。 |
| NoData 值 | 浮点数 | 否 | `0.0` | 要写入的 NoData 数值。 |

## 算法内容

NoData 设置用于给栅格波段写入无效值定义。当前实现支持两种模式：当波段号大于 0 时，只修改指定波段；当波段号小于等于 0 时，对全部波段一起写入同一个 NoData 值。该操作会直接修改原始文件，不生成新文件。

## 处理流程

1. 以可写方式打开输入栅格。
2. 读取目标波段号和要写入的 NoData 值。
3. 如果波段号小于等于 0，则遍历所有波段并逐个写入 NoData。
4. 如果波段号大于 0，则只对指定波段写入 NoData。
5. 原地保存修改并返回结果。

## 输出说明

- 本功能原地修改输入文件，不生成新的输出文件。
- 修改完成后可通过“栅格信息”查看对应波段的 NoData 值是否已更新。

## 结果解释

- 修改后，后续读取该栅格的流程可以把对应数值识别为无效像元。
- 操作不会改写像元数值本身，变化的是波段元数据中的 NoData 标记。

## 适用场景

- 栅格清洗、拼接前统一无效值口径和后续分析前的数据整理。

## 注意事项

- GUI 默认值是 `1`，所以默认只改第 1 波段。
- 如果希望全部波段一起设置，需要传 `0` 或负数。
