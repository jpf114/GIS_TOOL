# 栅格工具 / 伪彩色

## 功能说明

对单波段栅格应用颜色映射，输出 3 波段彩色结果。

## 输入与输出

- 输入：单波段或可指定单波段的栅格
- 输出：三波段彩色 GeoTIFF，建议 `.tif`

## 参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| 输入文件 | 文件路径 | 是 | 无 | 输入栅格。 |
| 输出文件 | 文件路径 | 是 | 无 | 输出彩色结果。 |
| 波段号 | 整数 | 否 | `1` | 从 1 开始。 |
| 颜色映射 | 枚举 | 否 | `jet` | `jet` `viridis` `hot` `cool` `spring` `summer` `autumn` `winter` `bone` `hsv` `rainbow` `ocean`。 |

## 算法内容

伪彩色用于把单波段数值栅格转换成更易观察的彩色影像。当前实现会先把指定波段归一化到 `0-255`，再套用 OpenCV 颜色映射，最终输出 3 波段彩色 GeoTIFF。

## 处理流程

1. 打开输入栅格并读取指定波段。
2. 统计该波段最小值和最大值。
3. 如果波段没有有效值域，则直接报错。
4. 将波段值归一化到 `0-255` 的 8 位范围。
5. 根据选择的颜色映射名称应用色带。
6. 将生成的彩色结果拆分为 3 个通道。
7. 创建新的 3 波段 GTiff 输出，并复制原始栅格的空间参考和地理变换。
8. 写出 3 个颜色通道，形成彩色结果。

## 输出说明

- 输出结果为栅格文件。
- 建议优先使用 `.tif` 保存，便于后续在 CLI、GUI 或 GIS 软件中继续处理。

## 结果解释

- 输出图像主要用于显示和辅助判读，不再保留原始单波段数值语义。
- 同一种数据在不同色带下会有不同视觉强调效果。

## 适用场景

- DEM、温度、指数或灰度栅格的可视化展示。

## 推荐输出文件名

- `伪彩色_result.tif`

## 注意事项

- 当波段没有有效值域时会失败。
- 当前实现固定输出 GTiff。

## 枚举值说明

### 颜色映射

- `jet`：彩虹色
- `viridis`：Viridis
- `hot`：热度色带
- `cool`：冷色带
- `spring`：春季色带
- `summer`：夏季色带
- `autumn`：秋季色带
- `winter`：冬季色带
- `bone`：灰白骨架色带
- `hsv`：HSV 色环
- `rainbow`：彩虹
- `ocean`：海洋
