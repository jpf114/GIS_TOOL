# 光谱指数 / 自定义指数

## 功能说明

根据用户提供的表达式或内置预设计算自定义光谱指数，适合快速试验业务口径或临时组合波段。

## 输入与输出
- 输入：多波段栅格
- 输出：自定义指数栅格，建议 `.tif`

## 参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| 输入文件 | 文件路径 | 是 | 无 | 输入多波段影像 |
| 输出文件 | 文件路径 | 是 | 无 | 输出自定义指数结果 |
| 表达式 | 字符串 | 否 | 空 | 自定义表达式，例如 `(B4-B3)/(B4+B3)` |
| 预设表达式 | 枚举 | 否 | `none` | 内置表达式预设 |
| 蓝光波段 | 整数 | 否 | `1` | `BLUE` 别名对应波段 |
| 绿光波段 | 整数 | 否 | `2` | `GREEN` 别名对应波段 |
| 红光波段 | 整数 | 否 | `3` | `RED` 别名对应波段 |
| 近红外波段 | 整数 | 否 | `4` | `NIR` 别名对应波段 |
| 短波红外1波段 | 整数 | 否 | `5` | `SWIR1` 别名对应波段 |

## 算法内容

自定义指数的目的不是固定实现某一个标准指数，而是给出一个可配置的表达式求值框架，让用户按业务口径直接组合波段。当前实现支持两种方式：一种是直接输入表达式，另一种是选择内置预设表达式。表达式里既可以用 `B1`、`B2` 这样的波段编号别名，也可以用 `BLUE`、`GREEN`、`RED`、`NIR`、`SWIR1` 这样的语义别名。

它本质上仍然是逐像元计算：对每个像元，把表达式里涉及到的波段值代入，再按四则运算和括号求值，最终生成一幅新的单波段指数图。

## 处理流程

1. 先确定本次实际使用的表达式：
   - 如果显式填写了 `expression`，优先使用该表达式。
   - 如果 `expression` 为空，则改用 `preset` 对应的内置表达式。
2. 打开输入栅格，并读取全部波段到内存。
3. 根据表达式中是否实际引用了 `BLUE`、`GREEN`、`RED`、`NIR`、`SWIR1` 等语义别名，解析这些别名应对应的波段号。
4. 逐像元构造表达式求值所需的变量集合，包括 `B1/B2/...` 这类编号别名，以及 `BLUE/GREEN/RED/NIR/SWIR1` 这类语义别名。
5. 对每个像元按表达式执行四则运算和括号求值，得到当前像元的指数值。
6. 将整幅结果写出为单波段浮点栅格，并记录表达式、预设和实际使用的波段信息。

## 输出说明

- 支持直接使用波段编号别名：`B1`、`B2`、`B3`
- 支持使用语义别名：`BLUE`、`GREEN`、`RED`、`NIR`、`SWIR1`
- 内置预设包括：
  - `ndvi_alias`
  - `ndmi_alias`
  - `ndwi_alias`
  - `mndwi_alias`
  - `ndbi_alias`
  - `bsi_alias`
  - `gndvi_alias`
  - `savi_alias`
  - `evi_alias`
  - `evi2_alias`

## 结果解释

- 输出结果含义完全取决于表达式本身，没有统一的物理解释。
- 如果使用的是内置预设，本质上就是把对应标准指数换成了统一表达式执行流程。
- 如果表达式涉及除法，当前流程会在分母极小的情况下按安全方式处理，避免明显异常数值直接破坏整图结果。

## 适用场景

- 临时验证新的指数口径或专题指标。
- 不想单独开发新算法，希望直接复用现有波段做组合计算。
- 在标准指数之外，快速试验业务自定义公式。

## 推荐输出文件名
- `custom_index_result.tif`

## 注意事项

- `expression` 为空时会自动使用 `preset`
- 如果表达式引用了 `BLUE`、`RED` 这类别名，需要同时保证对应波段号设置正确
- 当前表达式支持基础四则运算和括号
