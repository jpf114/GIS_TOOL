# 地形分析 / 视域分析

## 功能说明

以单个观察点为中心，计算 DEM 上的可视域范围。

## 输入与输出

- 输入：DEM 栅格
- 输出：视域结果栅格，建议 `.tif`

## 参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| 输入文件 | 文件路径 | 是 | 无 | DEM 输入 |
| 输出文件 | 文件路径 | 是 | 无 | 输出视域结果 |
| 波段号 | 整数 | 否 | `1` | DEM 波段 |
| observer_x | 浮点数 | 是 | 无 | 观察点 X 坐标 |
| observer_y | 浮点数 | 是 | 无 | 观察点 Y 坐标 |
| observer_height | 浮点数 | 否 | `2.0` | 观察点高度，必须大于等于 0 |
| target_height | 浮点数 | 否 | `0.0` | 目标高度，必须大于等于 0 |
| max_distance | 浮点数 | 否 | `0.0` | 最大分析距离，`0` 表示不限 |

## 算法内容

视域分析用于判断从单个观察点出发，DEM 上哪些位置在地形遮挡条件下可见。当前实现直接调用 GDAL 的 `ViewshedGenerate` 流程完成分析，因此结果语义与标准视域分析流程一致。

## 处理流程

1. 打开输入 DEM 并检查指定波段。
2. 读取观察点坐标、观察高度、目标高度和最大分析距离。
3. 调用 GDAL 视域分析流程，以当前波段作为高程输入。
4. 按观察点和地形遮挡关系计算每个像元是否可见。
5. 将生成的视域结果写出为单波段栅格。

## 输出说明

- 输出结果为栅格文件。
- 建议优先使用 `.tif` 保存，便于后续在 CLI、GUI 或 GIS 软件中继续处理。

## 结果解释

- 当前实现会生成标准视域结果栅格，可见区域与不可见区域对应不同像元值。
- 结果依赖 DEM 精度、观察点位置和高度设置。
- 如果设置了 `max_distance`，分析范围会被限制在该距离内。

## 适用场景

- 通视分析、站点布设、景观可视范围估计。

## 推荐输出文件名

- `terrain_viewshed_result.tif`

## 注意事项

- 建议输入具有明确高程含义的 DEM 数据。
- 涉及距离、面积或坡度解释时，应确认坐标系与高程单位一致。
