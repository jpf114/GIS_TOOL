# 地形分析 / TRI

## 功能说明

按 `3x3` 邻域计算地形崎岖度指数 TRI，用于衡量中心像元与周边高程差异强度。

## 输入与输出

- 输入：DEM 栅格
- 输出：TRI 结果栅格，建议 `.tif`

## 参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| 输入文件 | 文件路径 | 是 | 无 | DEM 输入 |
| 输出文件 | 文件路径 | 是 | 无 | 输出 TRI 栅格 |
| 波段号 | 整数 | 否 | `1` | DEM 波段 |
| z_factor | 浮点数 | 否 | `1.0` | 高程缩放系数 |

## 算法内容

TRI 用来衡量中心像元与周边邻域之间的地形起伏强度。当前实现使用固定 `3x3` 邻域，计算中心像元与周边 8 邻元高差的均方根，作为局部崎岖度近似值。

## 处理流程

1. 打开输入 DEM 并读取指定波段。
2. 按 `z_factor` 对高程进行缩放。
3. 对每个像元遍历 `3x3` 邻域中的 8 个相邻像元。
4. 计算每个邻元与中心像元的高差平方并累计。
5. 对累计结果取平均后再开平方，得到 TRI 结果。
6. 若存在 NoData，则恢复这些位置的 NoData 值。
7. 输出为单波段浮点栅格。

## 注意事项

- 当前实现按邻域高差均方根近似 TRI
- 结果越大通常表示地形越崎岖

## 输出说明

- 输出结果为单波段浮点栅格。
- 建议优先使用 `.tif` 保存，便于后续在 CLI、GUI 或 GIS 软件中继续处理。

## 结果解释

- 值越大，说明中心像元与周边高差整体越明显，局部地形越崎岖。
- 值越小，说明局部地形更平缓。

## 适用场景

- 山区崎岖度评价、地貌分区和地形复杂度分析。

## 推荐输出文件名

- `terrain_tri_result.tif`
