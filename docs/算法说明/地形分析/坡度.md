# 地形分析 / 坡度

## 功能说明

根据 DEM 计算坡度栅格。

## 输入与输出

- 输入：DEM 栅格
- 输出：坡度结果栅格，建议 `.tif`

## 参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| 输入文件 | 文件路径 | 是 | 无 | DEM 输入 |
| 输出文件 | 文件路径 | 是 | 无 | 输出坡度栅格 |
| 波段号 | 整数 | 否 | `1` | DEM 波段 |
| z_factor | 浮点数 | 否 | `1.0` | 高程缩放系数 |

## 算法内容

坡度用于描述地表在局部区域内的倾斜程度，是 DEM 最基础也最常用的派生地形指标之一。当前实现不是手工逐像元计算坡度，而是直接调用 GDAL 的 DEM 处理流程完成，因此其结果语义与标准 DEMProcessing 流程保持一致。

在这个流程里，坡度的核心含义是“地表高程在空间上的变化速率”。`z_factor` 用来调整高程方向的缩放比例，适合在水平单位与高程单位不一致、或希望放大/缩小垂向影响时使用。

## 处理流程

1. 打开输入 DEM 栅格，并确认参与计算的波段。
2. 将用户设置的波段号和 `z_factor` 传入 DEM 处理参数。
3. 调用 GDAL 的坡度计算流程，对指定波段进行地形导出。
4. 由 GDAL 根据 DEM 邻域高程变化计算坡度值。
5. 将计算结果直接写出为新的坡度栅格。

## 输出说明

- 输出结果为栅格文件。
- 建议优先使用 `.tif` 保存，便于后续在 CLI、GUI 或 GIS 软件中继续处理。

## 结果解释

- 输出栅格中的每个像元值表示该位置的坡度大小。
- 坡度越大，说明局部地表越陡；坡度越小，说明地表越平缓。
- `z_factor` 会直接影响结果幅度。如果高程单位和水平单位本来就一致，通常保持默认值即可；如果人为放大 `z_factor`，结果会表现得更陡。

## 适用场景

- 地形分级、坡耕地分析、工程选址等基础地形分析任务。
- 为山体阴影、视域分析、流域分析等后续地形派生过程提供参考。
- 与坡向、曲率等指标联合使用，描述地表形态。

## 推荐输出文件名

- `terrain_slope_result.tif`

## 注意事项

- 建议输入具有明确高程含义的 DEM 数据。
- 涉及距离、面积或坡度解释时，应确认坐标系与高程单位一致。
