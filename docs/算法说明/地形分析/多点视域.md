# 地形分析 / 多点视域

## 功能说明

以多个观察点为中心合并可视域范围。

## 输入与输出

- 输入：DEM 栅格
- 输出：合并后的视域结果栅格，建议 `.tif`

## 参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| 输入文件 | 文件路径 | 是 | 无 | DEM 输入 |
| 输出文件 | 文件路径 | 是 | 无 | 输出视域结果 |
| 波段号 | 整数 | 否 | `1` | DEM 波段 |
| observer_points | 字符串 | 是 | 无 | 观察点列表，格式如 `x1,y1;x2,y2` |
| observer_height | 浮点数 | 否 | `2.0` | 观察点高度 |
| target_height | 浮点数 | 否 | `0.0` | 目标高度 |
| max_distance | 浮点数 | 否 | `0.0` | 最大分析距离，`0` 表示不限 |

## 算法内容

多点视域用于合并多个观察点的可视范围，回答“这些观察点中至少有一个能看到哪些地方”。当前实现不是一次性做多点求解，而是对每个观察点分别运行单点视域分析，再把所有结果逐像元取并集。

## 处理流程

1. 打开输入 DEM 并检查指定波段。
2. 解析 `observer_points`，得到多个观察点坐标。
3. 初始化一幅全 0 的合并结果栅格。
4. 对每个观察点分别调用 GDAL 视域分析流程，生成单点视域结果。
5. 将每次单点结果与当前合并结果逐像元取最大值，形成并集。
6. 所有观察点处理完成后，将合并结果输出为单波段栅格。

## 输出说明

- 输出结果为栅格文件。
- 建议优先使用 `.tif` 保存，便于后续在 CLI、GUI 或 GIS 软件中继续处理。

## 结果解释

- 输出表示“至少被一个观察点看见”的综合视域结果。
- 观察点越多，合并后的可见范围通常越大。
- 当前结果不区分被几个观察点看到，只表达是否进入合并可视范围。

## 适用场景

- 多站点通视分析、监控点布设和联合可视覆盖评估。

## 推荐输出文件名

- `terrain_viewshed_multi_result.tif`

## 注意事项

- 建议输入具有明确高程含义的 DEM 数据。
- 涉及距离、面积或坡度解释时，应确认坐标系与高程单位一致。
