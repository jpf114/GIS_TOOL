# 地形分析 / 流向

## 功能说明

按 D8 规则计算每个像元的主流向编码。

## 输入与输出

- 输入：DEM 栅格
- 输出：流向编码栅格，建议 `.tif`

## 参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| 输入文件 | 文件路径 | 是 | 无 | DEM 输入 |
| 输出文件 | 文件路径 | 是 | 无 | 输出流向结果 |
| 波段号 | 整数 | 否 | `1` | DEM 波段 |
| z_factor | 浮点数 | 否 | `1.0` | 高程缩放系数 |

## 算法内容

流向用于确定每个像元的主下泄方向。当前实现采用 D8 规则：在 8 个相邻方向中，选择下降坡度最大的方向作为该像元的流向，并输出对应方向编码。

## 处理流程

1. 打开输入 DEM 并读取指定波段。
2. 按 `z_factor` 对高程进行缩放。
3. 构建 NoData 掩膜，避免无效像元参与计算。
4. 对每个有效像元检查 8 个邻接方向。
5. 计算中心像元到每个邻元的高差坡度。
6. 选取下降坡度最大的方向，并写入对应 D8 编码。
7. 若没有有效下游方向，则结果保持为 0。
8. 输出流向编码栅格。

## 输出说明

- 当前实现输出 D8 主流向编码

## 结果解释

- 输出值不是高程，而是方向编码。
- 编码表示水流优先流向的 8 邻方向，可作为汇流累积、河网和流域分析的基础输入。

## 适用场景

- 水文分析基础步骤。
- 汇流累积、河网提取和流域划分前置计算。

## 推荐输出文件名

- `terrain_flow_direction_result.tif`

## 注意事项

- 建议输入具有明确高程含义的 DEM 数据。
- 涉及距离、面积或坡度解释时，应确认坐标系与高程单位一致。
