# 地形分析 / 坡向

## 功能说明

根据 DEM 计算坡向栅格。

## 输入与输出

- 输入：DEM 栅格
- 输出：坡向结果栅格，建议 `.tif`

## 参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| 输入文件 | 文件路径 | 是 | 无 | DEM 输入 |
| 输出文件 | 文件路径 | 是 | 无 | 输出坡向栅格 |
| 波段号 | 整数 | 否 | `1` | DEM 波段 |
| z_factor | 浮点数 | 否 | `1.0` | 高程缩放系数 |

## 算法内容

坡向用于描述坡面朝向哪个方向，是 DEM 的基础派生指标之一。当前实现不手写逐像元坡向公式，而是直接调用 GDAL 的标准 DEM 处理流程完成，输出语义与常规 DEMProcessing 坡向结果保持一致。

## 处理流程

1. 打开输入 DEM 并检查指定波段是否存在。
2. 将波段号和 `z_factor` 组装为 DEM 处理参数。
3. 调用 GDAL 的坡向计算流程。
4. 由 GDAL 根据 DEM 邻域高程变化生成坡向结果。
5. 将结果写出为新的坡向栅格。

## 输出说明

- 输出结果为栅格文件。
- 建议优先使用 `.tif` 保存，便于后续在 CLI、GUI 或 GIS 软件中继续处理。

## 结果解释

- 输出像元值表示该位置坡面主朝向。
- 结果适合与坡度、山体阴影、太阳角度等一起联合分析地形受光和坡面形态。

## 适用场景

- 坡面朝向分析、太阳辐照分析和地形光照校正前处理。

## 推荐输出文件名

- `terrain_aspect_result.tif`

## 注意事项

- 建议输入具有明确高程含义的 DEM 数据。
- 涉及距离、面积或坡度解释时，应确认坐标系与高程单位一致。
