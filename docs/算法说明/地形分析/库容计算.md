# 地形分析 / 库容计算

## 功能说明

按指定水位计算 DEM 蓄水深度，并统计淹没面积与库容。

## 输入与输出

- 输入：DEM 栅格
- 输出：蓄水深度结果栅格，建议 `.tif`

## 参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| 输入文件 | 文件路径 | 是 | 无 | DEM 输入 |
| 输出文件 | 文件路径 | 是 | 无 | 输出蓄水深度栅格 |
| 波段号 | 整数 | 否 | `1` | DEM 波段 |
| water_level | 浮点数 | 是 | 无 | 目标水位 |

## 算法内容

库容计算用于在给定目标水位条件下，估算 DEM 表示地形中可形成的蓄水深度、淹没面积和总库容。当前实现直接用“水位减去地表高程”得到蓄水深度，并按像元面积对淹没区域做面积和体积累计。

## 处理流程

1. 打开输入 DEM 并检查指定波段。
2. 读取该波段高程数据。
3. 对每个像元计算 `water_level - elevation`。
4. 将小于 0 的结果截断为 0，得到蓄水深度栅格。
5. 根据栅格地理变换计算单个像元面积。
6. 对深度大于 0 的像元累计淹没面积。
7. 对深度大于 0 的像元按“深度 × 像元面积”累计库容。
8. 输出蓄水深度栅格，并在元数据中写入水位、淹没面积和库容。

## 输出说明

结果元数据中会附带：

- `reservoir_area`
- `reservoir_volume`
- `water_level`

## 结果解释

- 输出栅格表示各位置在当前水位下的蓄水深度。
- `reservoir_area` 表示被淹没像元累计得到的面积。
- `reservoir_volume` 表示各像元蓄水深度积分得到的总体积。

## 适用场景

- 水库方案比选、蓄水淹没分析和地形蓄水能力快速估算。

## 注意事项

- 当前输出栅格表示 `max(water_level - elevation, 0)`。
- `water_level` 应与 DEM 高程单位保持一致。
- 淹没面积与库容基于像元面积直接累计。

## 推荐输出文件名

- `terrain_reservoir_volume_result.tif`
