# 分类统计 / SVM 分类

## 功能说明

使用训练样本 CSV 对多波段栅格执行监督分类，当前采用 OpenCV SVM 的 RBF 核实现。

## 输入与输出

- 输入：
  - 输入栅格：建议 `.tif`
  - 训练样本 CSV：至少包含一个标签列和若干特征列
- 输出：
  - 分类结果栅格：建议 `.tif`

## 训练样本 CSV 格式

典型格式如下：

```csv
label,b1,b2
1,10,20
1,12,18
2,200,180
2,195,185
```

## 参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| 输入文件 | 文件路径 | 是 | 无 | 待分类栅格 |
| 输出文件 | 文件路径 | 是 | 无 | 输出分类结果 |
| 训练样本 CSV | 文件路径 | 是 | 无 | 监督分类样本文件 |
| 标签列 | 字符串 | 否 | `label` | 类别标签列名 |
| 波段列表 | 字符串 | 否 | 空 | 逗号分隔，留空表示使用全部波段 |

## 算法内容

该算法使用监督分类思路：先用训练样本学习“不同类别在多波段特征空间中的分布”，再把整幅影像中的每个像元映射到这个特征空间里，由模型判断它更接近哪个类别。当前实现使用的是 OpenCV 的 SVM 分类器，采用 `C_SVC` 模式和 `RBF` 核，适合处理类别边界并不完全线性的场景。

这里的训练样本 CSV 不是空间样本点文件，而是已经整理好的特征表。每一行代表一个训练样本，标签列表示类别编号，其余特征列表示参与训练的波段特征值。算法并不会自己从矢量样本中提取像元值，而是直接读取这张表进行训练。

## 处理流程

1. 读取训练样本 CSV，识别表头并定位标签列。
2. 将除标签列之外的列视为特征列，逐行解析出训练特征和类别标签。
3. 确定参与分类的波段列表；如果没有显式指定，则默认使用输入栅格全部波段。
4. 检查训练样本中的特征列数量是否与波段数量一致，不一致时直接报错。
5. 用训练样本构造 SVM 模型并完成训练。
6. 打开输入栅格，读取所有参与分类的波段数据。
7. 逐像元提取这些波段的值，拼成与训练样本同结构的特征向量。
8. 将每个像元特征向量送入训练好的 SVM 模型，得到预测类别。
9. 把预测类别写入单波段输出栅格，形成整幅分类图。

## 输出说明

- 输出结果为单波段分类栅格
- 像元值即训练样本中的类别编号

## 结果解释

- 输出栅格中的每个像元值，表示该像元被判定为哪个训练类别。
- 结果类别编码直接来自训练样本标签列，不会自动重编号或生成类别名称。
- 分类结果的可靠性高度依赖训练样本质量；如果样本数量偏少、类别分布不均衡或特征代表性不足，整图结果会明显受影响。

## 适用场景

- 已经具备训练样本特征表，希望快速生成监督分类结果。
- 多波段特征区分度较明显，且需要对整幅影像逐像元分类。
- 用作分类流程原型验证、样本质量验证和基础专题提取。

## 推荐输出文件名

- `svm_classify_result.tif`

## 注意事项

- 特征列数量必须与 `bands` 数量一致
- 训练样本至少应覆盖两个类别
