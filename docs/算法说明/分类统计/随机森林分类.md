# 分类统计 / 随机森林分类

## 功能说明

使用训练样本 CSV 对多波段栅格执行随机森林监督分类。

## 输入与输出

- 输入：
  - 输入栅格：建议 `.tif`
  - 训练样本 CSV：至少包含一个标签列和若干特征列
- 输出：
  - 分类结果栅格：建议 `.tif`

## 训练样本 CSV 格式

```csv
label,b1,b2
1,10,20
1,12,18
2,200,180
2,195,185
```

## 参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| 输入文件 | 文件路径 | 是 | 无 | 待分类栅格 |
| 输出文件 | 文件路径 | 是 | 无 | 输出分类结果 |
| 训练样本 CSV | 文件路径 | 是 | 无 | 监督分类样本文件 |
| 标签列 | 字符串 | 否 | `label` | 类别标签列名 |
| 波段列表 | 字符串 | 否 | 空 | 逗号分隔，留空表示使用全部波段 |

## 算法内容

随机森林分类是一种基于多棵决策树集成投票的监督分类方法。它通过训练样本学习“不同类别在多波段特征空间中的分布差异”，再让每个像元经过森林中的多棵树独立判断，最终按综合投票结果给出类别。当前实现使用 OpenCV 的随机森林分类器，适合做轻量但相对稳健的栅格监督分类。

与 SVM 类似，这里的训练样本输入是已经整理好的 CSV 特征表，而不是矢量样本图层。每一行训练数据都需要同时提供类别标签和对应波段特征值。

## 处理流程

1. 读取训练样本 CSV，识别标签列与特征列。
2. 解析全部训练记录，构造训练特征矩阵和类别标签。
3. 确定参与分类的波段列表；如果没有手动指定，则默认读取输入栅格全部波段。
4. 检查训练样本特征列数量是否与波段数量一致。
5. 用训练样本训练随机森林模型，建立类别判别规则。
6. 读取输入栅格中参与分类的各个波段。
7. 逐像元提取波段值，形成单个像元的特征向量。
8. 将该特征向量送入随机森林模型，得到预测类别。
9. 把预测类别写入单波段输出栅格，形成整幅分类结果。

## 输出说明

- 输出结果为单波段分类栅格
- 像元值即训练样本中的类别编号

## 结果解释

- 输出像元值就是模型预测的类别编号，编号来源于训练样本标签列。
- 随机森林对噪声样本和局部异常值通常比单一规则分类更稳一些，但结果仍然受训练样本代表性影响。
- 如果样本类别之间波段差异较弱，或者训练样本本身分布不均衡，结果仍可能出现混分。

## 适用场景

- 希望在现有样本表基础上快速得到监督分类图。
- 需要比单一线性判别更灵活的多波段分类方法。
- 做基础地物提取、样本验证和分类流程联调。

## 推荐输出文件名

- `random_forest_classify_result.tif`

## 注意事项

- 特征列数量必须与 `bands` 数量一致
- 当前实现适合轻量监督分类和流程联调
