# 分类统计 / 地物分类统计

## 功能说明

按面要素范围对多源分类栅格执行优先级统计，可输出统计表、分类面和分类栅格。

## 输入与输出

- 输入：
  - 输入面矢量：建议 `.gpkg` `.shp` `.geojson`
  - 分类映射：`.json`
  - 分类栅格列表：多个栅格路径，逗号分隔，建议 `.tif`
- 输出：
  - 统计输出：必填，仅支持 `.json` 或 `.csv`
  - 分类面输出：可选，当前仅支持 `.gpkg`
  - 分类栅格输出：可选，当前仅支持 `.tif` `.tiff`

## 配置文件格式

`分类映射` 必须是 JSON，典型格式如下：

```json
{
  "1": "耕地",
  "2": "林地",
  "3": "水体"
}
```

如果你的分类栅格像元值是 1、2、3、4，可以直接按这个结构改：

```json
{
  "1": "耕地",
  "2": "林地",
  "3": "水体",
  "4": "建设用地"
}
```

## 参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| 输入面矢量 | 文件路径 | 是 | 无 | 参与统计的面要素数据。 |
| 分类映射 | 文件路径 | 是 | 无 | 分类值到分类名称的 JSON 映射。 |
| 分类栅格列表 | 字符串 | 是 | 无 | 多个分类栅格路径，英文逗号分隔，顺序即优先级。 |
| 统计输出 | 文件路径 | 是 | 无 | 只支持 `.json` 或 `.csv`。 |
| 要素 ID 字段 | 字符串 | 否 | 无 | 面要素唯一标识字段。 |
| 要素名称字段 | 字符串 | 否 | 无 | 面要素名称字段。 |
| 波段列表 | 字符串 | 否 | 无 | 与 `分类栅格列表` 一一对应，逗号分隔，默认全部为 `1`。 |
| NoData 列表 | 字符串 | 否 | 无 | 与 `分类栅格列表` 一一对应，逗号分隔，默认全部为 `0`。 |
| 目标 EPSG | 整数 | 否 | `0` | 显式指定统计时使用的目标投影。 |
| 分类面输出 | 文件路径 | 否 | 无 | 可选矢量输出，当前仅支持 `.gpkg`。 |
| 分类栅格输出 | 文件路径 | 否 | 无 | 可选栅格输出，当前仅支持 `.tif` 或 `.tiff`。 |

## 算法内容

地物分类统计用于把多个分类栅格按统一空间网格投影到面要素范围内，再按照“栅格列表顺序即优先级”的规则，为每个面要素统计最终类别及面积。它不是简单地分别统计每张栅格，而是先把多源分类结果统一到同一目标坐标系、同一像元网格和同一面要素掩膜下，再逐像元决策最终类别。

当前实现会先读取分类映射、面要素和所有分类栅格，确定目标坐标系和目标网格；然后对每个面要素建立局部窗口，按窗口范围重采样读取所有栅格；对于落在面范围内的每个像元，按栅格输入顺序依次判断，只要遇到有效分类值就作为该像元最终分类。之后再汇总各类别像元数量和面积，并可选输出分类面或分类栅格。

## 填写示例

### 分类栅格列表 `rasters`

```text
D:/data/class_a.tif,D:/data/class_b.tif,D:/data/class_c.tif
```

### 波段列表 `bands`

```text
1,1,1
```

### NoData 列表 `nodatas`

```text
0,0,255
```

### 输出说明

- 主输出为统计结果文件，支持 `.json` 或 `.csv`。
- 如果填写了可选输出路径，还会额外生成分类面结果 `.gpkg` 和分类栅格结果 `.tif/.tiff`。

## 推荐输出文件名

统计输出：

```text
D:/data/scene_classification_feature_stats.json
```

分类面输出：

```text
D:/data/scene_classification_feature_stats.gpkg
```

分类栅格输出：

```text
D:/data/scene_classification_feature_stats.tif
```

## 处理流程

1. 解析输入参数，检查分类栅格列表、波段列表和 NoData 列表数量是否一致。
2. 读取分类映射 JSON，打开输入面矢量并读取要素 ID、名称及几何。
3. 检查所有输入栅格的空间参考、分辨率和范围信息。
4. 根据显式 `目标 EPSG` 或输入栅格情况，确定统计使用的目标坐标系。
5. 为所有输入栅格推导目标网格，并构建统一的统计网格参数。
6. 如果需要输出分类栅格，先根据所有面要素范围合并出全局输出窗口。
7. 逐个面要素处理：
8. 将面几何投影到目标坐标系，并计算其覆盖窗口。
9. 为当前面要素生成掩膜，只统计掩膜内像元。
10. 按窗口范围把每张分类栅格重采样到同一目标网格。
11. 按栅格输入顺序逐像元决策最终类别，前面的栅格优先级更高。
12. 统计各类别像元数、面积、占比，并补充要素标识信息。
13. 如果配置了分类面输出，则把当前要素范围内的分类结果写成面结果图层。
14. 如果配置了分类栅格输出，则把当前窗口结果写入全局分类栅格。
15. 汇总全部要素记录并写出 JSON 或 CSV 统计结果。

## 注意事项

- 当输入栅格全部为地理坐标系时，通常需要显式提供 `目标 EPSG`。
- `分类栅格列表`、`波段列表`、`NoData 列表` 都使用英文逗号分隔。
- `统计输出` 后缀不是 `.json` 或 `.csv` 会直接失败。
- `分类面输出` 当前不是 `.gpkg` 会直接失败。
- `分类栅格输出` 当前不是 `.tif` 或 `.tiff` 会直接失败。

## 结果解释

- 主统计结果会给出每个面要素在各类别上的像元数量、面积和占比。
- 当某个面要素在所有输入栅格里都没有有效分类值时，会写出空记录而不是强行指定类别。
- 分类面输出更适合直接浏览每个面要素内部的分类结果；分类栅格输出更适合后续继续做栅格分析。

## 适用场景

- 一个行政区、地块或地类样方内，综合多期或多来源分类结果做优先级统计。
- 需要给每个面要素汇总主导类别、面积和比例的专题统计任务。
- 需要同时产出统计表、分类面和分类栅格，供汇报、质检和后续分析使用。
