# 影像处理与分析 / GLCM 纹理

## 功能说明

按局部窗口计算灰度共生矩阵纹理特征，当前支持 `contrast`、`homogeneity`、`energy`、`entropy`。

## 输入与输出

- 输入：单波段或多波段栅格
- 输出：单波段浮点栅格，建议 `.tif`

## 参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| 输入文件 | 文件路径 | 是 | 无 | 输入影像 |
| 输出文件 | 文件路径 | 是 | 无 | 输出纹理结果 |
| 波段号 | 整数 | 否 | `1` | 参与处理的波段 |
| 核大小 | 整数 | 否 | `5` | 纹理窗口大小，必须为不小于 3 的奇数 |
| glcm_metric | 枚举 | 否 | `contrast` | 输出纹理指标 |
| glcm_levels | 整数 | 否 | `8` | 量化灰度级数，必须不小于 2 |

## 算法内容

GLCM 纹理用于描述局部邻域内灰度组合关系。它不是直接看单个像元值，而是统计一个窗口内“某种灰度与另一种灰度相邻出现”的频率，再由这个灰度共生矩阵推导出纹理特征。当前实现支持 `contrast`、`homogeneity`、`energy`、`entropy` 四种常见纹理指标。

这类结果更适合表达局部纹理结构，而不是原始亮度。输出值高低的含义会随所选纹理指标变化，因此使用时需要先明确当前要突出的是“差异感”“均匀性”“能量集中度”还是“复杂度”。

## 处理流程

1. 读取指定波段，并转换为 8 位灰度数据。
2. 根据 `glcm_levels` 将原始灰度量化到较少的灰度级，降低共生矩阵规模。
3. 以 `kernel_size` 定义局部滑动窗口，在窗口内部逐对统计水平方向相邻像元的灰度共现次数。
4. 将共现次数归一化为概率分布，形成当前窗口的灰度共生矩阵。
5. 根据 `glcm_metric` 计算当前窗口的纹理值：
   - `contrast` 强调灰度差异
   - `homogeneity` 强调灰度接近程度
   - `energy` 强调概率集中程度
   - `entropy` 强调纹理复杂度
6. 将每个窗口中心位置得到的纹理值写入结果栅格。
7. 对整幅结果做归一化，输出到 `0` 到 `1` 范围内的单波段浮点栅格。

## 输出说明

- 输出结果会归一化到 `0` 到 `1`
- 不同 `glcm_metric` 对应不同纹理含义

## 结果解释

- `contrast` 值高，通常表示邻域灰度变化更明显，纹理更粗或边界更强。
- `homogeneity` 值高，通常表示邻域灰度更一致，区域更平滑。
- `energy` 值高，通常表示某些灰度组合更集中，纹理模式更稳定。
- `entropy` 值高，通常表示局部纹理更复杂、杂乱程度更高。
- 由于窗口边界处无法完整构造邻域，边缘区域通常会较少参与有效计算。

## 适用场景

- 纹理分区、目标识别前的特征提取。
- 区分平滑区、粗糙区、规则纹理区和复杂纹理区。
- 作为后续分类、分割或变化分析的辅助特征层。

## 推荐输出文件名

- `glcm_texture_result.tif`

## 注意事项

- `kernel_size` 必须为奇数
- `glcm_levels` 不能小于 2
