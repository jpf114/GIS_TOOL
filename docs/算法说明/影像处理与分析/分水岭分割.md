# 影像处理与分析 / 分水岭分割

## 功能说明

执行分水岭分割，支持外部标记文件，也支持自动生成标记。

## 输入与输出

- 输入：
  - 输入影像
  - 可选标记文件
- 输出：分割标签栅格，建议 `.tif`

## 参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| 输入文件 | 文件路径 | 是 | 无 | 输入影像。 |
| 输出文件 | 文件路径 | 是 | 无 | 输出分割结果。 |
| 波段号 | 整数 | 否 | `1` | 从 1 开始。 |
| 标记输入 | 文件路径 | 否 | 无 | 外部分割标记；留空则自动生成。 |

## 算法内容

分水岭分割把灰度影像看成地形表面，通过标记种子区域从低处向外扩展，最后在分界处形成边界。当前实现既支持外部提供标记图，也支持根据阈值、距离变换和连通组件自动生成标记。

## 处理流程

1. 读取输入影像指定波段，并转换为 8 位灰度图。
2. 如果提供了 `标记输入`，则直接读取标记图并转为整型标记矩阵。
3. 如果没有提供标记图，则先对灰度图做 Otsu 二值化。
4. 对二值结果进行膨胀，得到较稳的背景区域。
5. 对前景做距离变换，并按最大距离的一半阈值提取较稳的前景区域。
6. 对前景执行连通组件标记，生成初始种子。
7. 将前景与背景之间的未知区域标记为 0。
8. 把灰度图转为三通道图像并执行分水岭分割。
9. 将分割结果转换为标签栅格：边界写为 `-1`，有效分区写为正标签值。

## 输出内容

- 元数据返回 `segment_count`。

## 标记输入要求

如果提供 `标记输入`，应满足：

- 是一幅栅格文件
- 能和输入影像对应到同一空间范围和尺寸
- 其像元值可被转换为整型标记图

常见约定：

- `0`：背景或未标记区域
- `1, 2, 3...`：不同的种子区域编号

简单示意：

```text
0 0 0 0 0
0 1 1 0 2
0 1 1 0 2
0 0 0 0 2
```

上面表示存在两个种子区域，编号分别为 `1` 和 `2`。

如果不提供，代码会自动根据阈值、距离变换和连通域生成标记。

## 输出说明

- 输出结果通常是单波段标签栅格，建议 `.tif`。
- 元数据中的 `segment_count` 表示分割后标签数量。

## 结果解释

- 输出不是分类图，而是分割标签图。
- 相同标签表示属于同一个分割区域，`-1` 表示分水岭边界。

## 适用场景

- 目标分割、区域生长前处理和标记分区任务。

## 推荐输出文件名

- `分水岭分割_result.tif`

## 注意事项

- 建议先确认输入波段号、数据类型和 NoData 设置是否正确。
- 输出结果受输入影像质量和参数设置影响较大。
