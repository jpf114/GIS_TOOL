# 影像处理与分析 / 连通组件

## 功能说明

对二值目标执行连通组件标记，为每个连通区域分配唯一标签值。

## 输入与输出
- 输入：栅格影像
- 输出：连通组件标签栅格，建议 `.tif`

## 参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| 输入文件 | 文件路径 | 是 | 无 | 输入栅格影像 |
| 输出文件 | 文件路径 | 是 | 无 | 输出标签栅格 |
| 波段 | 整数 | 否 | `1` | 参与计算的输入波段 |

## 算法内容

连通组件用于把二值前景中的每个独立连通区域赋予唯一编号。当前实现会先对输入影像做 Otsu 二值化，再按 8 邻域执行连通组件标记，最终输出标签栅格。

## 处理流程

1. 读取输入影像指定波段。
2. 将波段转为 8 位灰度图。
3. 对灰度图执行 Otsu 二值化，得到前景和背景。
4. 按 8 邻域执行连通组件标记。
5. 将整型标签结果转为浮点栅格输出。
6. 在元数据中返回有效连通区域数量。

## 输出说明

- 输出像元值表示所属连通区域编号
- 不同连通区域具有不同标签值

## 结果解释

- 0 通常表示背景。
- 大于 0 的不同标签值表示不同前景连通区域。

## 适用场景

- 连通区域计数、目标分区和前景区域编号。

## 推荐输出文件名
- `connected_components_result.tif`

## 注意事项

- 更适合输入为二值栅格
- 如果前景噪声较多，建议先做滤波或形态学清理
