# 影像处理与分析 / Gabor 滤波

## 功能说明

按给定方向和尺度对单波段栅格执行 Gabor 滤波，常用于纹理响应提取。

## 输入与输出

- 输入：单波段或多波段栅格
- 输出：单波段浮点栅格，建议 `.tif`

## 参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| 输入文件 | 文件路径 | 是 | 无 | 输入影像 |
| 输出文件 | 文件路径 | 是 | 无 | 输出滤波结果 |
| 波段号 | 整数 | 否 | `1` | 参与处理的波段 |
| 核大小 | 整数 | 否 | `9` | Gabor 核大小，必须为正奇数 |
| sigma | 浮点数 | 否 | `2.0` | 高斯包络尺度 |
| gabor_theta | 浮点数 | 否 | `0.0` | 滤波方向角，单位为弧度 |
| gabor_lambda | 浮点数 | 否 | `8.0` | 滤波波长，必须大于 0 |
| gabor_gamma | 浮点数 | 否 | `0.5` | 核纵横比，必须大于 0 |
| gabor_psi | 浮点数 | 否 | `0.0` | 相位偏移 |

## 算法内容

Gabor 滤波本质上是用一个同时包含方向性和频率信息的卷积核去扫描影像局部区域，从而突出与指定方向、指定纹理尺度相匹配的结构响应。当前实现面向单个处理波段工作，不直接做多尺度组合或多方向融合，而是一次生成一个指定参数下的响应结果。

从使用角度看，这个算法更适合回答“这个波段里，某个方向和某个纹理尺度上的结构有多明显”这类问题。输出不是分类结果，也不是边界线，而是一幅纹理响应强度图，数值越高，表示局部区域越符合当前 Gabor 核所表达的纹理模式。

## 处理流程

1. 打开输入栅格，并读取指定波段的数据。
2. 将该波段转换为浮点格式，保证后续卷积与归一化计算稳定。
3. 根据 `kernel_size`、`sigma`、`gabor_theta`、`gabor_lambda`、`gabor_gamma`、`gabor_psi` 构造 Gabor 卷积核。
4. 用该卷积核对整幅波段执行二维滤波，得到每个像元在当前方向和尺度下的响应值。
5. 对响应结果做最小值到最大值归一化，将结果压缩到统一数值范围，便于后续显示、比较和阈值处理。
6. 将归一化后的结果写出为单波段浮点栅格。

## 输出说明

- 输出结果会做归一化，范围通常落在 `0` 到 `1`
- 输出主要反映局部方向纹理响应强弱

## 结果解释

- 值较高的区域，通常表示局部纹理在方向、间距或条带模式上更接近当前设置的 Gabor 核。
- 值较低的区域，通常表示该区域缺少对应纹理，或纹理方向、尺度与当前参数不匹配。
- 如果改变 `gabor_theta`，会强调不同方向的纹理结构；如果改变 `gabor_lambda` 和 `sigma`，会改变对纹理粗细和尺度的敏感程度。
- 由于结果已经做过归一化，不同任务之间的数值不建议直接拿来做物理量比较，更适合在同一次处理结果中观察相对强弱。

## 适用场景

- 提取条带状、方向性明显的纹理响应。
- 为后续分割、目标检测或纹理分析提供辅助特征。
- 对比不同方向参数下的响应结果，分析影像中的主导纹理方向。

## 推荐输出文件名

- `gabor_filter_result.tif`

## 注意事项

- `kernel_size` 需要为正奇数
- `gabor_lambda` 和 `gabor_gamma` 必须大于 0
