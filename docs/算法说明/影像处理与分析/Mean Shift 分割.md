# 影像处理与分析 / Mean Shift 分割

## 功能说明

按空间邻域和灰度邻域执行 Mean Shift 平滑分割，适合做灰度影像的区域平滑与粗分割。

## 输入与输出

- 输入：单波段或多波段栅格
- 输出：单波段浮点栅格，建议 `.tif`

## 参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| 输入文件 | 文件路径 | 是 | 无 | 输入影像 |
| 输出文件 | 文件路径 | 是 | 无 | 输出分割结果 |
| 波段号 | 整数 | 否 | `1` | 参与处理的波段 |
| spatial_radius | 浮点数 | 否 | `10.0` | 空间邻域半径，必须大于 0 |
| color_radius | 浮点数 | 否 | `20.0` | 灰度邻域半径，必须大于 0 |
| pyramid_level | 整数 | 否 | `1` | 金字塔层级，必须不小于 0 |

## 算法内容

Mean Shift 分割的核心思想是把空间位置相近、灰度值相近的像元向局部密集区域不断聚合，从而实现平滑和区域化。当前实现采用 OpenCV 的均值漂移滤波流程，对单波段灰度数据做区域平滑，输出的是平滑后的灰度分割结果，而不是带独立区域编号的标签图。

因此，这里的“分割”更准确地说是“区域平滑后的粗分区表达”。它适合先把噪声和细碎变化压下去，让同类区域更加连贯，再交给后续阈值、轮廓或分类流程继续处理。

## 处理流程

1. 读取指定波段数据。
2. 将原始数据转换为 8 位灰度影像。
3. 为适配均值漂移滤波接口，把单通道灰度临时扩展成三通道影像。
4. 以 `spatial_radius` 控制空间邻域范围，以 `color_radius` 控制灰度相似性范围，执行 Mean Shift 滤波。
5. 将滤波后的结果再转回灰度图。
6. 把灰度结果归一化到 `0` 到 `1` 左右的浮点范围，并写出为单波段栅格。

## 输出说明

- 输出结果为归一化后的灰度分割结果
- 更大的半径会带来更强的平滑效果

## 结果解释

- 输出结果不是“类别编号图”，而是经过区域平滑后的灰度结果图。
- `spatial_radius` 越大，空间上更远的像元也更容易被合并到同一区域。
- `color_radius` 越大，灰度差异更大的区域也更容易被平滑为相近结果。
- 如果参数过大，细节和边界会被明显抹平；如果参数过小，则分割和平滑效果不明显。

## 适用场景

- 预处理阶段的区域平滑和粗分区。
- 在轮廓提取、阈值分割前先压制局部噪声。
- 希望保留较大结构、削弱细碎纹理时使用。

## 推荐输出文件名

- `mean_shift_segment_result.tif`

## 注意事项

- `spatial_radius` 和 `color_radius` 必须大于 0
- `pyramid_level` 不能小于 0
