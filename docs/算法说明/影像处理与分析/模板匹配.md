# 影像处理与分析 / 模板匹配

## 功能说明

使用模板影像在输入影像中查找最佳匹配位置，并输出匹配响应图。

## 输入与输出

- 输入：
  - 输入影像
  - 模板文件
- 输出：匹配响应栅格，建议 `.tif`

## 参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| 输入文件 | 文件路径 | 是 | 无 | 搜索影像。 |
| 输出文件 | 文件路径 | 是 | 无 | 输出匹配响应图。 |
| 波段号 | 整数 | 否 | `1` | 从 1 开始。 |
| 模板文件 | 文件路径 | 是 | 无 | 模板影像。 |
| 匹配方法 | 枚举 | 否 | `ccoeff_normed` | `sqdiff` `sqdiff_normed` `ccorr` `ccorr_normed` `ccoeff` `ccoeff_normed`。 |

## 算法内容

模板匹配用于在输入影像中搜索与模板影像最相似的位置。当前实现针对指定波段进行灰度模板匹配，输出的是整幅匹配响应图，同时返回最佳匹配位置和得分。

## 处理流程

1. 读取输入影像和模板影像指定波段。
2. 将两者统一转为 8 位灰度图。
3. 检查模板尺寸是否大于输入影像，若大于则直接报错。
4. 根据所选匹配方法确定 OpenCV 模板匹配模式。
5. 在整幅输入影像上执行模板匹配，生成响应图。
6. 从响应图中找出最佳匹配位置和对应得分。
7. 将响应图输出为单波段栅格，并把最佳匹配位置写入元数据。

## 输出说明

- 输出结果为栅格文件。
- 建议优先使用 `.tif` 保存，便于后续在 CLI、GUI 或 GIS 软件中继续处理。

## 结果解释

- 输出栅格不是“是否匹配”的二值图，而是整幅匹配响应图。
- 对于相关性类方法，值越大通常越好；对于平方差类方法，值越小通常越好。

## 适用场景

- 小目标定位、重复结构搜索和模板位置验证。

## 推荐输出文件名

- `模板匹配_result.tif`

## 注意事项

- 模板尺寸不能大于输入影像。
- 元数据包含最佳匹配位置和得分。

## 模板文件要求

- 模板文件本身就是一幅普通栅格影像
- 应与输入影像来自相近传感器或同类数据
- 模板尺寸必须小于等于输入影像

### 填写示例

- 输入文件：`D:/data/source_scene.tif`
- 模板文件：`D:/data/source_scene_template.tif`
- 输出文件：`D:/out/template_match_response.tif`

## 枚举值说明

### 匹配方法

- `sqdiff`：平方差，值越小越好
- `sqdiff_normed`：归一化平方差，值越小越好
- `ccorr`：相关性，值越大越好
- `ccorr_normed`：归一化相关性，值越大越好
- `ccoeff`：相关系数，值越大越好
- `ccoeff_normed`：归一化相关系数，值越大越好
