# 投影转换 / 重投影

## 功能说明

将输入数据重投影到目标坐标系。代码同时支持栅格和矢量输入，输出格式由输出文件后缀决定。

## 输入与输出

- 输入：
  - 栅格：常用 `.tif` `.tiff` `.img`
  - 矢量：常用 `.shp` `.gpkg` `.geojson` `.json` `.kml` `.csv`
- 输出：
  - 栅格建议使用 `.tif`
  - 矢量建议使用 `.gpkg`、`.geojson`、`.shp`
- 输出文件必填

## 参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| 输入文件 | 文件路径 | 是 | 无 | 输入栅格或矢量数据路径。 |
| 输出文件 | 文件路径 | 是 | 无 | 输出路径，后缀决定输出格式。 |
| 目标坐标系 | 坐标系 | 是 | 无 | 例如 `EPSG:3857`，也可传 WKT。 |
| 源坐标系 | 坐标系 | 否 | 无 | 留空时优先使用输入数据自带坐标系。 |
| 重采样方式 | 枚举 | 否 | `nearest` | 栅格重投影时生效，可选 `nearest` `bilinear` `cubic` `cubicspline` `lanczos` `average` `mode`。 |

## 算法内容

重投影用于把空间数据真正转换到新的目标坐标系。当前实现同时支持栅格和矢量两条处理路径：如果输入能按矢量打开，则走矢量重投影；否则按栅格重投影处理。也就是说，它不是只改标签，而是会按目标坐标系生成新的实际输出数据。

## 处理流程

1. 读取输入文件、输出文件、目标坐标系、可选源坐标系和可选重采样方式。
2. 先尝试把输入作为矢量数据打开。
3. 如果是矢量数据：
4. 根据输出文件后缀自动判断目标格式。
5. 必要时清理已有旧输出。
6. 组装矢量重投影参数，并调用 GDALVectorTranslate 执行坐标转换和写出。
7. 如果不是矢量数据，则按栅格处理：
8. 打开输入栅格，组装栅格重投影参数。
9. 按目标坐标系和重采样方式调用 GDALWarp 执行重投影。
10. 将新的重投影结果写出到输出文件。

## 枚举值说明

### 重采样方式

- `nearest`：最近邻，速度快，适合分类栅格
- `bilinear`：双线性，适合连续型栅格
- `cubic`：三次卷积，结果更平滑
- `cubicspline`：三次样条
- `lanczos`：高质量重采样，速度相对更慢
- `average`：均值重采样
- `mode`：众数重采样，适合离散分类数据

## 处理逻辑

- 如果输入能按矢量打开，就走矢量重投影流程。
- 否则按栅格重投影处理。
- 矢量输出格式由输出后缀自动识别：
  - `.shp` -> `ESRI Shapefile`
  - `.gpkg` -> `GPKG`
  - `.kml` -> `KML`
  - `.csv` -> `CSV`
  - 其他默认按 `GeoJSON`

## 结果解释

- 栅格重投影会生成新的像元网格，结果受重采样方式影响。
- 矢量重投影会把几何坐标转换到目标坐标系，输出文件格式由后缀决定。
- 如果填写了 `src_srs`，会优先用它覆盖输入数据自带的源坐标系理解。

## 适用场景

- 栅格和矢量数据统一坐标系。
- 数据交换、叠加分析和发布前的空间参考标准化。

## 输出说明

- 输出为重投影后的实际数据文件。
- 栅格输出建议使用 `.tif`，矢量输出建议使用 `.gpkg`、`.geojson` 或 `.shp`。

## 推荐输出文件名

- `重投影_result.json`
- `重投影_result.csv`
- `重投影_result.gpkg`
- `重投影_result.geojson`
- `重投影_result.shp`
- `重投影_result.tif`

## 注意事项

- `目标坐标系` 不能为空。
- 栅格重投影时，输出一般应使用 `.tif`。
- 矢量写出前会删除同名旧输出，尤其是 Shapefile 会连带删同名前缀的侧车文件。
