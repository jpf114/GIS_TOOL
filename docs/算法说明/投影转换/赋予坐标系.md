# 投影转换 / 赋予坐标系

## 功能说明

直接给输入栅格写入坐标系定义，不做重采样，也不改变像元值与像素位置。

## 输入与输出

- 输入：栅格文件
- 输出：原地修改输入文件，不生成新文件

## 参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| 输入文件 | 文件路径 | 是 | 无 | 要写入坐标系的栅格文件。 |
| 坐标系 | 坐标系 | 是 | 无 | 例如 `EPSG:4326`，也可传 WKT。 |

## 算法内容

赋予坐标系的作用是给已有栅格直接写入空间参考定义，而不改变像元值、分辨率、像素位置和 GeoTransform。当前实现本质上是“写入投影标签”，不是重新采样，也不是实际坐标变换。

## 处理流程

1. 读取输入栅格路径和目标坐标系定义。
2. 解析输入的坐标系参数。
3. 将坐标系对象导出为 WKT。
4. 以可写方式打开输入栅格。
5. 直接把新的投影 WKT 写入数据集。
6. 原地保存并返回结果，不生成新文件。

## 输出说明

- 本功能原地修改输入文件，不生成新的输出文件。
- 修改完成后可直接再次使用“坐标信息”或“栅格信息”检查写入结果。

## 结果解释

- 处理后数据会携带新的坐标系定义。
- 由于没有重采样，像元内容和像素布局不变，变化的只是空间参考标签。

## 适用场景

- 数据本身坐标系明确，但文件缺少或写错了投影标签。
- 为后续重投影、叠加分析补齐空间参考信息。

## 注意事项

- 这是“写标签”，不是“重投影”。
- 如果原数据实际坐标系与填写内容不一致，后续空间分析会出错。
- 当前是原地修改输入文件，因此不会生成新的输出文件，也没有输出后缀可选。
