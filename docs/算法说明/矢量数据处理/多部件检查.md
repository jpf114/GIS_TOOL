# 矢量数据处理 / 多部件检查

## 功能说明

检查多部件要素并导出 CSV 结果表。

## 输入与输出
- 输入：矢量数据
- 输出：CSV 检查结果

## 参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| 输入文件 | 文件路径 | 是 | 无 | 输入矢量文件 |
| 输出文件 | 文件路径 | 是 | 无 | 输出 CSV 文件 |
| 图层 | 字符串 | 否 | 空 | 指定输入图层 |

## 算法内容

当前实现不限制点、线、面类型，而是按几何对象是否属于多部件结构来检查。它会统计每个要素的部件数量，并把部件数量大于 1 的要素输出到 CSV。

判断逻辑有两层：一是几何实际部件数大于 1；二是几何类型本身就是 `MultiPoint`、`MultiLineString`、`MultiPolygon` 或 `GeometryCollection`。满足任一条件就会被报告。

## 处理流程

1. 打开输入图层。
2. 创建 CSV 输出并写入表头。
3. 逐要素读取几何。
4. 统计当前几何的部件数量。
5. 判断几何类型是否属于多部件类型。
6. 对多部件要素输出 `source_fid,geom_type,part_count`。
7. 返回发现数量和扫描数量。

## 输出说明

- 输出文件固定为 CSV。
- `geom_type` 记录输入几何类型名称。
- `part_count` 表示该要素实际包含的部件个数。
- 只有命中的多部件要素会写出，普通单部件要素不会出现在结果中。

## 推荐输出文件名

- `多部件检查_result.csv`

## 注意事项

- 该工具只检查，不拆分几何。
- `GeometryCollection` 也会被视为多部件结构。
- 空几何的部件数按 0 处理，不会作为多部件问题输出。

## 结果解释

- `part_count` 大于 1 说明一个要素内部包含多个独立部件。
- 如果结果为空，说明没有检测到多部件要素。
- `scanned_count` 表示实际扫描的总要素数。

## 适用场景

- 入库前筛查不符合“单要素单对象”要求的数据。
- 为“转单部件”处理做预检查。
- 质检多源合并后的混合几何结果。
