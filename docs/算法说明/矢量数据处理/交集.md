# 矢量数据处理 / 交集

## 功能说明

提取输入图层与叠加图层共同覆盖的空间部分。

## 输入与输出

- 输入：
  - 输入矢量图层
  - 叠加矢量图层
- 输出：交集结果图层，建议使用 `.gpkg`

## 参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| 输入文件 | 文件路径 | 是 | 无 | 主输入图层 |
| 输出文件 | 文件路径 | 是 | 无 | 交集结果输出路径 |
| 图层名 | 字符串 | 否 | 无 | 输入图层名 |
| 叠加矢量 | 文件路径 | 是 | 无 | 参与交集的第二个图层，默认取其第一个图层 |

## 算法内容

程序会先把叠加图层几何标准化，必要时尝试自动修复非法几何。普通情况下，对每个输入要素与叠加几何求空间交集；若输入是线、叠加是面，则先按叠加面边界切分线段，再只保留位于叠加面内的线段。输出只保留输入图层字段，不保留叠加图层字段。

## 处理流程

1. 打开输入图层和叠加图层。
2. 检查两者 CRS 是否一致。
3. 读取并标准化叠加图层几何。
4. 若叠加几何数量较少，先合并为整体叠加几何；否则按输入要素构造局部候选叠加几何。
5. 创建输出图层并复制输入字段。
6. 逐条处理输入要素：
   - 线 + 面时，先切分线，再保留位于叠加面内部的线段；
   - 其它情况直接计算交集几何。
7. 将非空交集写出。

## 结果解释

- 输出几何表示输入要素与叠加要素真正重合的空间部分
- 未相交的输入要素不会出现在结果中
- 输出属性全部来自输入图层
- 返回数量是成功写出的交集结果要素数

## 适用场景

- 提取某区域内的道路、河流、地块等要素
- 找出两个专题图层真正重合的部分
- 从线要素中提取落在指定面范围内的有效片段

## 输出说明

- 输出图层名称固定为 `intersect`
- 输出字段沿用输入图层
- 结果几何类型以输入图层有效类型为主

## 推荐输出文件名

- `intersect_result.gpkg`

## 注意事项

- 输入图层与叠加图层必须使用相同 CRS
- 叠加图层中的非法几何会先尝试自动修复
- 当前实现只保留输入字段，不会把叠加图层字段一并写出
