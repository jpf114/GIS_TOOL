# 矢量数据处理 / 格式转换

## 功能说明

把输入矢量图层复制到指定目标格式，不改变几何和字段内容。

## 输入与输出

- 输入：矢量图层
- 输出：目标格式矢量文件，支持 `GeoJSON`、`ESRI Shapefile`、`GPKG`、`KML`、`CSV`

## 参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| 输入文件 | 文件路径 | 是 | 无 | 输入矢量数据 |
| 输出文件 | 文件路径 | 是 | 无 | 目标文件路径 |
| 图层名 | 字符串 | 否 | 无 | 目标图层名；留空时取默认图层 |
| 输出格式 | 枚举 | 否 | `GeoJSON` | 目标驱动格式 |

## 算法内容

该功能本质上是一次图层复制。程序按 `输出格式` 选择驱动，创建目标图层后，把输入图层的字段结构、几何类型、空间参考和要素内容逐条写出。它不会做投影转换、字段映射或几何修复，重点只是把当前图层换一个存储格式。

## 处理流程

1. 打开输入矢量数据并定位目标图层。
2. 根据 `输出格式` 选择目标驱动。
3. 清理已存在的同名输出，特别是 `Shapefile` 的配套文件。
4. 创建输出数据集。
5. 以输入图层的原始名称、几何类型和空间参考创建输出图层。
6. 复制字段结构。
7. 逐条读取输入要素并写入输出图层。

## 结果解释

- 输出几何与输入保持一致
- 输出字段与输入保持一致
- 空间参考会直接复制，不做转换
- 返回消息中的数量是实际转换写出的要素数

## 适用场景

- 在不同软件或平台之间交换矢量数据
- 将中间结果统一转成 GeoPackage 便于后续处理
- 把多图层工作流中的单图层导出为 GeoJSON、KML 或 CSV
- 在发布前把结果转换成目标系统要求的格式

## 输出说明

- 结果格式由 `输出格式` 参数决定，不由输出后缀自动推断
- 输出图层名称沿用输入图层名称
- 目标格式驱动不支持的字段或几何限制由底层驱动处理

## 推荐输出文件名

- `格式转换_result.gpkg`
- `格式转换_result.geojson`
- `格式转换_result.shp`
- `格式转换_result.kml`
- `格式转换_result.csv`

## 注意事项

- 当前实现以 `输出格式` 参数选择驱动，因此 `输出格式` 与文件后缀最好保持一致
- 该功能不做坐标系转换；若需要换投影，应使用重投影功能
- 输入中若存在目标驱动不支持的能力，可能在写出时被底层驱动限制
