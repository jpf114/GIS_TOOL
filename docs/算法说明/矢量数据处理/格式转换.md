# 矢量数据处理 / 格式转换

## 功能说明

将矢量数据转换到指定格式。

## 输入与输出

- 输入：矢量文件
- 输出：目标格式矢量文件

## 参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| 输入文件 | 文件路径 | 是 | 无 | 输入矢量。 |
| 输出文件 | 文件路径 | 是 | 无 | 输出文件路径。 |
| 图层名 | 字符串 | 否 | 无 | 留空时取默认图层。 |
| 输出格式 | 枚举 | 否 | `GeoJSON` | `GeoJSON` `ESRI Shapefile` `GPKG` `KML` `CSV`。 |

## 输出说明

- 输出结果为目标格式矢量文件。
- 建议根据目标驱动选择匹配的文件后缀，避免生成结果和预期格式不一致。

## 推荐输出文件名

- `格式转换_result.csv`
- `格式转换_result.gpkg`
- `格式转换_result.kml`
- `格式转换_result.geojson`
- `格式转换_result.shp`

## 注意事项

- `输出格式` 和 `输出文件后缀` 最好保持一致。
- 如果 `输出格式` 与输出后缀不一致，实际生成结果可能与用户预期不一致，建议两者始终配套填写。

## 枚举值说明

### 输出格式

- `GeoJSON`：GeoJSON
- `ESRI Shapefile`：Shapefile
- `GPKG`：GeoPackage
- `KML`：KML
- `CSV`：CSV

## 推荐后缀对应关系

- `GeoJSON`：建议 `.geojson` 或 `.json`
- `ESRI Shapefile`：必须使用 `.shp`
- `GPKG`：建议 `.gpkg`
- `KML`：建议 `.kml`
- `CSV`：建议 `.csv`

## 填写示例

- 输入文件：`D:/data/parcels.shp`
- 输出文件：`D:/out/parcels.gpkg`
- 输出格式：`GPKG`
