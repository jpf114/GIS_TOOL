# 矢量数据处理 / 简化

## 功能说明

按给定容差对输入几何做拓扑保持简化，减少顶点数量。

## 输入与输出

- 输入：线或面矢量图层
- 输出：简化后的矢量图层

## 参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| 输入文件 | 文件路径 | 是 | 无 | 输入矢量数据 |
| 输出文件 | 文件路径 | 是 | 无 | 简化结果输出路径 |
| 图层名 | 字符串 | 否 | 无 | 目标图层名 |
| 容差 | 浮点数 | 否 | `10.0` | 简化容差，单位与图层坐标单位一致 |

## 算法内容

程序对每个要素调用 `SimplifyPreserveTopology`，也就是在减小顶点数量的同时尽量保持原有拓扑关系。字段结构和空间参考原样复制，输出几何类型保持与输入图层声明类型一致。若任何一个要素简化后为空或失败，当前实现会直接中止整个任务。

## 处理流程

1. 打开输入图层并检查坐标参考是否为投影坐标系。
2. 根据输出路径创建结果图层。
3. 复制字段结构。
4. 逐条读取输入要素。
5. 对要素几何执行拓扑保持简化。
6. 把简化后的几何和原字段写入输出图层。
7. 返回处理数量和本次使用的容差。

## 结果解释

- 输出几何通常比输入更少顶点、更平滑
- 字段内容不变
- `feature_count` 表示成功写出的要素数
- `tolerance` 表示实际使用的简化容差

## 适用场景

- 降低大比例尺数据的顶点复杂度
- 为前端展示或网络传输生成轻量版本
- 在不希望明显破坏拓扑的前提下做几何压缩

## 输出说明

- 输出图层保留输入字段结构和空间参考
- 输出几何类型沿用输入图层声明类型

## 推荐输出文件名

- `simplify_result.gpkg`

## 注意事项

- 当前实现要求输入图层是投影坐标系
- 容差必须大于 `0`
- 若某个要素简化失败或得到空几何，当前实现会直接报错结束，不会跳过该要素
