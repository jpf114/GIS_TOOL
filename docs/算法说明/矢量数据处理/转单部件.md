# 矢量数据处理 / 转单部件

## 功能说明

将多部件要素拆分为多个单部件要素，并保留原有属性字段。

## 输入与输出
- 输入：矢量数据
- 输出：单部件矢量数据

## 参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| 输入文件 | 文件路径 | 是 | 无 | 输入矢量文件 |
| 输出文件 | 文件路径 | 是 | 无 | 输出矢量文件 |
| 图层 | 字符串 | 否 | 空 | 指定输入图层 |

## 算法内容

当前实现支持点、线、面三类几何的单部件拆分。程序会递归展开 `MultiPoint`、`MultiLineString`、`MultiPolygon` 和 `GeometryCollection`，把其中每个最小单体几何分别复制成一条输出要素。

输出图层的几何类型会按输入主类型降成单部件类型，例如多面输出为面，多线输出为线。原属性字段会在每个拆分结果上完整复制，因此一条多部件记录可能变成多条属性相同、几何不同的记录。

## 处理流程

1. 打开输入图层并识别输入主几何类型。
2. 推导输出图层几何类型为点、线或面单体。
3. 创建输出图层并复制空间参考、字段结构。
4. 逐要素读取几何并做标准化。
5. 对空几何直接跳过。
6. 递归遍历多部件或集合几何，提取每个单部件几何。
7. 对每个单部件复制原属性并写入一条输出记录。
8. 返回输出数量、源要素数量和跳过数量。

## 输出说明

- 输出仍是矢量图层，但几何会被拆成单部件。
- `feature_count` 表示输出后的要素数。
- `source_count` 表示处理过的源要素数。
- 一条输入记录可能对应多条输出记录。

## 推荐输出文件名

- `转单部件_result.gpkg`

## 注意事项

- 只支持点、线、面三类几何，不支持其他复杂类型。
- 输出会增加要素数量，FID 也会重新分配。
- 原属性会重复复制到每个拆分结果上。

## 结果解释

- 当输出要素数明显大于源要素数时，说明图层里存在较多多部件对象。
- 如果某条记录被拆分成多个结果，这些结果在属性上相同，几何上各自独立。
- `skipped_count` 表示空几何被跳过的数量。

## 适用场景

- 面向只接受单部件要素的下游系统做预处理。
- 为面积、长度、邻接等逐几何分析做准备。
- 清理多源合并后形成的多部件记录。
