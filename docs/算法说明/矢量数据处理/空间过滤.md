# 矢量数据处理 / 空间过滤

## 功能说明

按属性条件、空间范围，或两者组合，从输入图层中筛出满足条件的要素并写出新图层。

## 输入与输出

- 输入：矢量文件
- 输出：过滤后的矢量文件，常用 `.gpkg`、`.geojson`、`.shp`、`.kml`

## 参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| 输入文件 | 文件路径 | 是 | 无 | 输入矢量数据 |
| 输出文件 | 文件路径 | 是 | 无 | 过滤结果输出路径 |
| 图层名 | 字符串 | 否 | 无 | 目标图层名；留空时取默认图层 |
| 属性过滤 | 字符串 | 否 | 无 | SQL `WHERE` 条件表达式 |
| 空间范围 | 范围 | 否 | `0,0,0,0` | `xmin,ymin,xmax,ymax` |

## 算法内容

该功能先在源图层上设置属性过滤条件，再设置矩形空间过滤范围，最后把当前过滤结果中的要素逐条复制到新图层。输出图层沿用原图层的几何类型、字段结构和空间参考，不重算几何，也不改变字段值。

## 处理流程

1. 打开输入数据集并定位目标图层。
2. 检查是否至少提供了属性过滤或空间范围之一。
3. 若填写属性过滤，则把 SQL 条件应用到图层。
4. 若填写空间范围，则对图层设置矩形空间过滤。
5. 根据输出后缀选择驱动；未识别时默认按 `GeoJSON` 输出。
6. 创建输出数据集和输出图层，并复制字段结构。
7. 读取当前过滤结果中的要素，逐条写入输出图层。

## 结果解释

- 输出中只保留同时满足当前过滤条件的要素
- 原始字段会完整保留
- 原始几何不被修改，只是按条件筛选
- 返回消息中的数量是实际写入输出图层的要素数

## 适用场景

- 按属性条件提取目标要素
- 从大范围数据中裁出一个矩形区域内的要素
- 先按属性再按空间范围缩小数据规模
- 为后续缓冲、叠加、统计等操作准备子集数据

## 输出说明

- 输出图层沿用输入图层的字段结构和空间参考
- 输出格式由输出文件后缀决定：`.shp`、`.gpkg`、`.kml` 分别走对应驱动，其它后缀默认按 `GeoJSON`

## 推荐输出文件名

- `filtered_result.gpkg`
- `空间过滤结果.geojson`

## 注意事项

- `属性过滤` 与 `空间范围` 至少填写一个，否则实现会直接失败
- 当前实现的执行顺序是先应用属性过滤，再应用空间范围过滤
- 若填写了图层名但数据集中只有一个图层，当前实现仍可能回退到该唯一图层
