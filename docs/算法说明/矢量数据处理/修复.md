# 矢量数据处理 / 修复

## 功能说明

尝试修复输入图层中的非法几何，并把可修复结果写出到新图层。

## 输入与输出

- 输入：矢量图层
- 输出：修复后的矢量图层

## 参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| 输入文件 | 文件路径 | 是 | 无 | 输入矢量数据 |
| 输出文件 | 文件路径 | 是 | 无 | 修复结果输出路径 |
| 图层名 | 字符串 | 否 | 无 | 目标图层名 |

## 算法内容

程序逐条读取输入要素几何。若几何本身有效，则直接复制；若无效，则调用几何标准化逻辑尝试修复，本质上会走 `MakeValid`。若某个要素几何为空、缺失或修复失败，则该要素不会写入结果，而是计入跳过数量。字段结构和空间参考会原样保留。

## 处理流程

1. 打开输入图层。
2. 创建输出图层并复制字段结构。
3. 逐条读取输入要素。
4. 对几何执行标准化：
   - 有效几何直接复制；
   - 非法几何尝试自动修复；
   - 空几何或修复失败则跳过。
5. 把修复后的几何和原字段写入结果图层。
6. 返回修复数量和跳过数量。

## 结果解释

- `repaired_count`：成功写出到结果图层的要素数量
- `skipped_count`：由于没有可用几何或修复失败而被跳过的要素数量
- 输出字段与输入一致
- 输出并不保证“所有问题都被修复”，只表示当前实现能写出的部分已经被接受

## 适用场景

- 叠加分析前先清理非法面
- 导入外部矢量数据后做一次快速几何修复
- 为拓扑检查、缓冲、融合等操作准备更稳定的输入

## 输出说明

- 输出图层保留输入字段结构和空间参考
- 只写出修复成功或原本有效的要素

## 推荐输出文件名

- `repair_result.gpkg`

## 注意事项

- 当前实现不会保留失败要素的占位记录，失败要素会被直接跳过
- 结果消息中的修复数量包含“原本有效直接复制”的要素
- 若需要区分“原本有效”和“真正调用修复后有效”的要素，当前实现没有单独字段或统计
