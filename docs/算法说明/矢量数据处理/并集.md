# 矢量数据处理 / 并集

## 功能说明

把输入图层与叠加图层做并集分析，输出覆盖二者空间范围的结果几何。

## 输入与输出

- 输入：
  - 输入矢量图层
  - 叠加矢量图层
- 输出：并集结果图层，建议使用 `.gpkg`

## 参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| 输入文件 | 文件路径 | 是 | 无 | 主输入图层 |
| 输出文件 | 文件路径 | 是 | 无 | 并集结果输出路径 |
| 图层名 | 字符串 | 否 | 无 | 输入图层名 |
| 叠加矢量 | 文件路径 | 是 | 无 | 第二个参与并集的图层，默认取其第一个图层 |

## 算法内容

并集前，叠加图层几何会先做标准化处理，必要时尝试自动修复非法几何。普通情况按“输入几何与候选叠加几何做 `Union`”处理；当输入是线、叠加是面时，当前实现走的是更实用的混合维度逻辑：先按叠加面边界把线切成多个片段，再把这些片段写出，而不是生成带叠加属性的传统 GIS 并集结果。输出中不保留输入属性字段，只新增一个 `source` 字段标记来源。

## 处理流程

1. 打开输入图层和叠加图层。
2. 检查两者 CRS 是否一致。
3. 读取并标准化叠加图层几何。
4. 若是“线 + 面”的混合维度并集，则判断叠加面是否高度重叠，过度重叠时直接拒绝执行。
5. 创建输出图层，并新增 `source` 字段。
6. 逐条处理输入要素：
   - 线 + 面时先切分线段并写出；
   - 普通情况若相交，则写出并集几何；
   - 若不相交，则直接写出原输入几何。

## 结果解释

- 输出几何覆盖输入与叠加图层共同涉及的空间范围
- `source=input`：该结果直接来自未与叠加几何相交的输入要素
- `source=union`：该结果来自真正的并集运算
- `source=split`：该结果来自线被面边界切分后的片段
- 当前实现不保留输入属性字段，也不写出叠加图层字段

## 适用场景

- 快速构造输入层与叠加层的合并覆盖范围
- 处理线状要素与面状范围的混合维度分段结果
- 在不关心原字段，只关心空间结果时生成统一结果图层

## 输出说明

- 输出图层名称固定为 `union`
- 输出字段固定包含 `source`
- 输出格式建议使用 `.gpkg` 或 `.geojson`

## 推荐输出文件名

- `union_result.gpkg`

## 注意事项

- 输入图层与叠加图层必须使用相同 CRS
- 叠加图层中的非法几何会先尝试自动修复；修复失败时整个操作失败
- 当前实现与传统属性并集不同：不会同时保留两边字段
- 对“线 + 面”混合维度并集，若叠加面高度重叠，实现会直接拒绝执行
