# 矢量数据处理 / 空间连接

## 功能说明

按空间相交关系，把连接图层的统计结果附加到输入图层要素上。

## 输入与输出

- 输入：
  - 输入矢量图层
  - 连接矢量图层
- 输出：附带连接统计字段的新矢量图层

## 参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| 输入文件 | 文件路径 | 是 | 无 | 主输入图层 |
| 输出文件 | 文件路径 | 是 | 无 | 结果输出路径 |
| 图层名 | 字符串 | 否 | 无 | 输入图层名 |
| 连接图层 | 文件路径 | 是 | 无 | 参与空间连接的目标图层，默认取其第一个图层 |
| 连接字段 | 字符串 | 否 | 无 | 指定时把相交要素该字段的去重值汇总写入结果 |

## 算法内容

当前实现的空间关系固定为 `Intersects`。程序对每个输入要素遍历连接图层全部要素，统计与其相交的目标要素数量；若指定连接字段，则把这些相交要素的该字段值去重后，用分号拼接写入 `join_value`。输出保留输入图层原始字段，不展开成一对多记录。

## 处理流程

1. 打开输入图层和连接图层。
2. 检查两者 CRS 是否一致。
3. 若指定连接字段，则检查连接图层中是否存在该字段。
4. 创建输出图层并复制输入字段。
5. 新增 `join_count` 字段；若指定连接字段，再新增 `join_value`。
6. 对每个输入要素遍历连接图层全部要素，按 `Intersects` 判断是否相交。
7. 统计相交数量，并把去重后的连接字段值拼接写入结果。

## 结果解释

- `join_count`：与当前输入要素相交的目标要素数量
- `join_value`：相交目标要素指定字段的去重汇总值，以 `;` 分隔
- 结果保持“一条输入要素对应一条输出要素”，不会把一对多关系展开成多行

## 适用场景

- 统计每个地块、行政单元或缓冲区内命中的目标要素数量
- 为输入要素补充相交对象的名称、类别等汇总信息
- 做轻量的空间命中统计，而不是完整字段拼接

## 输出说明

- 输出保留输入图层原始字段
- 另外新增 `join_count`，以及可选的 `join_value`
- 返回消息中的 `feature_count` 是实际处理并写出的输入要素数

## 推荐输出文件名

- `spatial_join_result.gpkg`

## 注意事项

- 输入图层与连接图层必须使用相同 CRS
- 当前空间关系固定为相交，不支持包含、相离、最近邻等其它关系
- 当前实现不会把连接图层的全部字段逐项展开到结果中，只写计数和可选的字段值汇总
