# 矢量数据处理 / 拓扑检查

## 功能说明

检查空几何、无效几何、重复面和重叠面，并导出 CSV 问题清单。

## 输入与输出
- 输入：面矢量数据
- 输出：CSV 拓扑问题表

## 参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| 输入文件 | 文件路径 | 是 | 无 | 输入矢量文件 |
| 输出文件 | 文件路径 | 是 | 无 | 输出 CSV 文件 |
| 图层 | 字符串 | 否 | 空 | 指定输入图层 |

## 算法内容

当前实现是一个面图层诊断工具，只支持面和多面图层，并要求使用投影坐标系。它会先逐要素检查几何是否为空、是否无效，然后再做两两比较，识别重复几何和面积重叠。

输出结果统一写入 CSV。对于单要素问题，会只填写 `source_fid`；对于成对问题，会同时填写 `source_fid` 和 `target_fid`。`issue_type` 目前实际只会出现 `empty_geometry`、`invalid_geometry`、`duplicate_geometry`、`overlap` 这四类。

## 处理流程

1. 打开输入图层并检查几何类型是否为面或多面。
2. 检查图层空间参考是否为投影坐标系。
3. 逐要素读取原始几何，记录 FID、是否为空、是否无效，并缓存标准化几何。
4. 对空几何输出 `empty_geometry`。
5. 对无效几何输出 `invalid_geometry`。
6. 对全部有效缓存几何做两两比较。
7. 如果两几何完全相等，输出 `duplicate_geometry`。
8. 否则如果两几何相交，再计算交集面积。
9. 当交集面积大于 0 时，输出 `overlap` 及其面积值。

## 输出说明

- 输出文件固定为 CSV，不是修复后的矢量图层。
- 表头固定为 `source_fid,target_fid,issue_type,issue_value`。
- `issue_value` 对空几何、无效几何、重复几何固定写 `0`。
- 对 `overlap`，`issue_value` 写实际重叠面积。

## 推荐输出文件名

- `拓扑检查_result.csv`

## 注意事项

- 只支持面图层。
- 该工具只报告问题，不会自动修复。
- “重复面”按几何完全相等判断，不比较属性字段是否一致。

## 结果解释

- `empty_geometry` 表示要素没有可用几何。
- `invalid_geometry` 表示原始几何拓扑不合法。
- `duplicate_geometry` 表示两要素几何完全一致。
- `overlap` 表示两要素存在真实面积重叠，`issue_value` 是重叠面积。

## 适用场景

- 入库前的数据体检。
- 修复工具执行前的定位诊断。
- 面状成果交付前的质量核查。
