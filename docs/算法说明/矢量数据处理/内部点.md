# 矢量数据处理 / 内部点

## 功能说明

提取面要素内部点为点图层，保持原有属性字段。

## 输入与输出
- 输入：面矢量数据
- 输出：内部点点图层

## 参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| 输入文件 | 文件路径 | 是 | 无 | 输入矢量文件 |
| 输出文件 | 文件路径 | 是 | 无 | 输出点图层 |
| 图层 | 字符串 | 否 | 空 | 指定输入图层 |

## 算法内容

当前实现只支持面和多面图层，但它实际采用的是 `Centroid()` 来生成点，而不是调用专门的“保证落面内”算法。因此从代码逻辑看，它更接近“按面计算中心点”，并不严格保证每个结果点一定在原面内部。

程序会逐要素读取标准化几何，尝试计算质心点，成功后复制原属性写入输出。空几何或无法得到质心的要素会被跳过。

## 处理流程

1. 打开输入图层并检查几何类型是否为面或多面。
2. 创建输出点图层，并复制空间参考和属性字段。
3. 逐要素读取几何并做标准化。
4. 对空几何直接跳过。
5. 调用 `Centroid()` 计算点位。
6. 把原属性字段和结果点写入输出图层。
7. 返回输出数量和跳过数量。

## 输出说明

- 输出是点图层。
- 原属性字段保留。
- 每条有效面要素对应一个点。
- 当前实现下，该点来自质心计算结果。

## 推荐输出文件名

- `内部点_result.gpkg`

## 注意事项

- 虽然名称是“内部点”，但当前实现并不严格保证点一定落在面内。
- 对凹多边形、环状面等复杂情况，结果点可能落在面外。
- 如果业务必须保证点在面内，当前实现需要额外核验结果。

## 结果解释

- 输出点可以作为面要素的中心代表点使用。
- 当结果点落在面外时，不是数据错误，而是当前实现方式导致的正常现象。
- `skipped_count` 表示空几何或质心计算失败的要素数量。

## 适用场景

- 面要素中心点快速提取。
- 标注、概览展示、近似代表点生成。
- 对“必须在面内”要求不严格的场景。
