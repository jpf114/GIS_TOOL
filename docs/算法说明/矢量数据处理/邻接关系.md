# 矢量数据处理 / 邻接关系

## 功能说明

分析同一面图层中要素之间的邻接关系，并导出 CSV 邻接表。

## 输入与输出
- 输入：面矢量数据
- 输出：CSV 邻接关系表

## 参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| 输入文件 | 文件路径 | 是 | 无 | 输入矢量文件 |
| 输出文件 | 文件路径 | 是 | 无 | 输出 CSV 文件 |
| 图层 | 字符串 | 否 | 空 | 指定输入图层 |

## 算法内容

当前实现只支持面或多面图层，并且要求图层坐标系是投影坐标系。程序会先把每个要素的几何做标准化复制，然后两两比较要素之间是否 `Touches`。只有两个面在边界上接触时，才会继续计算它们边界交集的长度。

输出不是邻接面，也不会修改原图层，而是生成一张 CSV 关系表。表中每一行代表一对相邻要素，记录源要素 FID、目标要素 FID，以及两者共享边界长度 `shared_length`。

## 处理流程

1. 打开输入图层并检查几何类型是否为面或多面。
2. 检查空间参考是否为投影坐标系；如果是经纬度图层会直接失败。
3. 读取全部非空几何并缓存其 FID 与标准化后的几何对象。
4. 对缓存要素做两两组合检查。
5. 先用 `Touches` 判断两要素是否接触，不接触则跳过。
6. 对接触要素分别取边界，再求边界交集。
7. 如果交集结果是线或多线，则计算其总长度并写入 CSV。
8. 输出 `source_fid,target_fid,shared_length` 三列结果。

## 输出说明

- 输出文件固定为 CSV，不是矢量图层。
- `source_fid` 和 `target_fid` 是输入图层中的要素 FID。
- `shared_length` 表示两要素共享边界的长度，单位跟输入图层坐标单位一致。
- 结果只包含检测到邻接关系的要素对，不会输出未相邻的要素。

## 推荐输出文件名

- `邻接关系_result.csv`

## 注意事项

- 只支持面图层，不支持点或线图层。
- 建议在米制等投影坐标系下使用，否则长度没有实际意义。
- 结果按要素对输出，不会生成对称重复记录。

## 结果解释

- 当 `shared_length` 大于 0 时，说明两要素存在实际共享边界。
- 如果只在角点接触，边界交集通常不是线，结果里不会形成有效共享长度。
- 结果条数越多，说明图层内部邻接关系越复杂。

## 适用场景

- 行政区、地块、宗地图层的邻接关系整理。
- 面要素拓扑网络分析前的关系表构建。
- 分区传播、邻接约束建模、空间图结构构建。
