# 矢量数据处理 / 顶点提取

## 功能说明

提取要素顶点为点图层，并附带来源要素和顶点序号。

## 输入与输出
- 输入：矢量数据
- 输出：顶点点图层

## 参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| 输入文件 | 文件路径 | 是 | 无 | 输入矢量文件 |
| 输出文件 | 文件路径 | 是 | 无 | 输出点图层 |
| 图层 | 字符串 | 否 | 空 | 指定输入图层 |

## 算法内容

当前实现支持点、线、面及其多部件集合。程序会递归遍历几何，把所有可直接读取的坐标点展开成点图层输出。对面要素，会提取外环和内环上的全部顶点；对线要素，会提取线串全部节点；对点要素，则直接把原点写出。

输出时会在原属性之外新增 `source_fid` 和 `vertex_idx` 两个字段，用来标识顶点来自哪条要素，以及在该要素顶点序列中的顺序。

## 处理流程

1. 打开输入图层并创建输出点图层。
2. 复制原字段结构，并新增 `source_fid`、`vertex_idx`。
3. 逐要素读取几何并做标准化。
4. 对空几何直接跳过。
5. 递归遍历点、线、面及多部件集合，收集全部顶点。
6. 按收集顺序逐点写出结果，并记录顶点序号。
7. 返回输出顶点数、源要素数和跳过数。

## 输出说明

- 输出是点图层。
- `source_fid` 表示顶点所属输入要素 FID。
- `vertex_idx` 表示当前顶点在该要素提取序列中的编号，从 0 开始。
- 面要素闭合环上的重复收尾点也会按当前实现被写出。

## 推荐输出文件名

- `顶点提取_result.gpkg`

## 注意事项

- 对面要素会同时提取外环和内环顶点。
- 多部件几何会把各部件顶点按遍历顺序连续编号。
- 如果不希望保留闭合环末尾重复点，需要在后处理阶段自行去重。

## 结果解释

- 输出点数量越多，说明源几何越复杂或顶点越密。
- `vertex_idx` 只在单条源要素内部有意义，不跨要素连续。
- `source_count` 是处理过的输入要素数，不等于输出点数。

## 适用场景

- 线面节点质检。
- 顶点级别导出、抽样、匹配前预处理。
- 需要把几何节点转成独立点要素进行后续分析。
