# 矢量数据处理 / 重叠检查

## 功能说明

分析同一面图层中存在面积重叠的要素对，并导出 CSV 结果表。

## 输入与输出
- 输入：面矢量数据
- 输出：CSV 检查结果

## 参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| 输入文件 | 文件路径 | 是 | 无 | 输入矢量文件 |
| 输出文件 | 文件路径 | 是 | 无 | 输出 CSV 文件 |
| 图层 | 字符串 | 否 | 空 | 指定输入图层 |

## 算法内容

当前实现只支持面或多面图层，并且要求图层使用投影坐标系。程序会先缓存所有非空几何，再逐对判断两要素是否相交；只有真正存在面面积交叠时，才会输出结果。

这里的“重叠”不是简单的边界接触。实现会对相交要素求交集几何，并且只在交集结果是面或多面且面积大于 0 时，才认定为有效重叠。输出表中记录的是要素对及其重叠面积。

## 处理流程

1. 打开输入图层并检查几何类型是否为面或多面。
2. 检查空间参考是否为投影坐标系。
3. 读取全部非空几何，缓存 FID 和标准化后的几何。
4. 对缓存要素做两两组合检查。
5. 先用 `Intersects` 判断是否有相交可能。
6. 对相交要素求交集几何。
7. 如果交集是面或多面，则计算其面积。
8. 仅当面积大于 0 时，输出 `source_fid,target_fid,overlap_area`。

## 输出说明

- 输出文件固定为 CSV，不是矢量图层。
- `overlap_area` 表示两个要素交叠部分的面积，单位跟输入图层坐标单位一致。
- 只会输出真正存在面积重叠的要素对。
- 纯边界接触、点接触或交集面积为 0 的情况不会写入结果。

## 推荐输出文件名

- `重叠检查_result.csv`

## 注意事项

- 只支持面图层。
- 建议在投影坐标系下使用，否则面积值不适合直接解读。
- 该工具只做检查，不会自动裁剪或修复重叠部分。

## 结果解释

- `overlap_area` 越大，说明两要素重叠范围越大。
- 如果结果为空，表示没有检测到面积重叠的要素对。
- 同一对要素只会输出一条记录，不会正反各写一次。

## 适用场景

- 地块、规划面、行政区等数据的质量检查。
- 合并数据源后排查重复入库或边界覆盖问题。
- 修复前先量化重叠问题规模。
