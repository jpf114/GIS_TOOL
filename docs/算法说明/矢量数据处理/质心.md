# 矢量数据处理 / 质心

## 功能说明

按要素生成质心点，并保留原有属性字段。

## 输入与输出
- 输入：矢量数据
- 输出：质心点图层

## 参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| 输入文件 | 文件路径 | 是 | 无 | 输入矢量文件 |
| 输出文件 | 文件路径 | 是 | 无 | 输出点图层 |
| 图层 | 字符串 | 否 | 空 | 指定输入图层 |

## 算法内容

当前实现按要素独立计算质心。程序会逐要素读取几何，做标准化后调用 `Centroid()` 生成点结果，再把输入要素的属性字段复制到输出点图层。

这里计算的是几何质心，不会额外检查质心是否落在原面内部；对于凹多边形或形状特殊的几何，输出点可能位于几何外部。

## 处理流程

1. 打开输入图层，识别输出格式并创建点图层。
2. 复制输入图层的坐标系和属性字段结构。
3. 逐要素读取输入几何并做标准化。
4. 对空几何直接跳过。
5. 对有效几何调用 `Centroid()` 计算质心点。
6. 复制原属性字段并写出质心点。
7. 返回成功输出数量和跳过数量。

## 输出说明

- 输出是点图层，不保留原几何类型。
- 原属性字段会完整保留。
- 每条有效输入要素对应一个质心点。
- 空几何不会输出。

## 推荐输出文件名

- `质心_result.gpkg`

## 注意事项

- 质心不保证一定落在原面内部。
- 如果需要保证点位在面内，当前更适合使用“内部点”。
- 对线和点图层也可计算，但输出统一是点。

## 结果解释

- 结果点表示几何的中心位置，不代表面积权重之外的业务中心。
- `skipped_count` 大于 0 表示部分输入几何为空。
- 复杂凹形面如果出现质心落面外，属于当前实现的正常表现。

## 适用场景

- 面要素标注点生成。
- 要素中心位置提取。
- 后续做最近邻、空间聚合、中心点展示。
