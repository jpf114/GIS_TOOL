# 矢量数据处理 / 凸包

## 功能说明

按要素生成最小凸包面，并保留原有属性字段。

## 输入与输出
- 输入：矢量数据
- 输出：凸包结果矢量数据

## 参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| 输入文件 | 文件路径 | 是 | 无 | 输入矢量文件 |
| 输出文件 | 文件路径 | 是 | 无 | 输出矢量文件 |
| 图层 | 字符串 | 否 | 空 | 指定输入图层 |

## 算法内容

当前实现按“单个要素对应一个凸包结果”的方式工作，不会把整层要素合成一个总凸包。程序会逐要素读取几何，先做标准化，再调用 GDAL/OGR 的 `ConvexHull()` 生成最小凸包面，并把原属性字段拷贝到输出图层。

空几何或标准化后为空的要素不会报错中断，而是直接跳过，并在结果元数据里累计 `skipped_count`。

## 处理流程

1. 打开输入图层，确定输出格式并创建目标图层。
2. 复制输入图层的空间参考和属性字段结构。
3. 逐要素读取输入几何并做标准化处理。
4. 对空几何直接跳过。
5. 对有效几何调用 `ConvexHull()` 生成凸包面。
6. 保留原属性字段，把凸包几何写入输出图层。
7. 输出完成后返回成功要素数和跳过要素数。

## 输出说明

- 输出仍然是面图层。
- 一条输入要素最多对应一条输出要素。
- 原属性字段会保留，但几何会变成凸包面。
- 如果部分输入几何为空，它们不会出现在输出中。

## 推荐输出文件名

- `凸包_result.gpkg`

## 注意事项

- 凸包结果会抹平凹陷和内凹边界。
- 该工具不会把多要素整体融合成一个公共凸包。
- 输出面范围通常大于或等于原几何范围。

## 结果解释

- 如果输入是线或点，输出也会被包成最小凸包面。
- 对原本已经接近凸形的面，输出变化通常较小。
- `skipped_count` 大于 0 时，说明有空几何被忽略。

## 适用场景

- 快速生成要素外包范围。
- 做目标范围概化、包络分析、粗粒度占地判断。
- 后续作为搜索范围、初筛范围的面结果。
