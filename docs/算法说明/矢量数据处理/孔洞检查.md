# 矢量数据处理 / 孔洞检查

## 功能说明

检查面要素内环孔洞并导出 CSV 结果表。

## 输入与输出
- 输入：面矢量数据
- 输出：CSV 检查结果

## 参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| 输入文件 | 文件路径 | 是 | 无 | 输入矢量文件 |
| 输出文件 | 文件路径 | 是 | 无 | 输出 CSV 文件 |
| 图层 | 字符串 | 否 | 空 | 指定输入图层 |

## 算法内容

当前实现只支持面和多面图层。程序会逐要素统计内部环数量：单面直接取内环数，多面则累加所有子面的内环数。只要内环数量大于 0，就认为该要素存在孔洞并写入结果。

这里检查的是几何结构上的内环数量，不区分这些孔洞是否合理，也不会计算孔洞面积或位置。

## 处理流程

1. 打开输入图层并检查几何类型是否为面或多面。
2. 创建 CSV 输出并写入表头。
3. 逐要素读取原始几何。
4. 统计当前几何内部环数量。
5. 当 `hole_count > 0` 时，输出 `source_fid,geom_type,hole_count`。
6. 返回命中数量和扫描数量。

## 输出说明

- 输出文件固定为 CSV。
- `geom_type` 是输入几何类型名称。
- `hole_count` 是该要素内部环总数。
- 只有存在孔洞的要素会写入结果。

## 推荐输出文件名

- `孔洞检查_result.csv`

## 注意事项

- 只支持面图层。
- 当前实现只统计孔洞数量，不输出孔洞几何本身。
- 孔洞不一定是错误，需结合业务判断。

## 结果解释

- `hole_count=1` 表示有一个内环，`2` 表示有两个内环，以此类推。
- 如果结果为空，说明没有检测到带孔洞的面要素。
- 多面要素会把全部子面的孔洞数累加统计。

## 适用场景

- 地块、行政区、水域等面成果质检。
- 判断是否存在意外空洞或保留洞区。
- 修复或制图前先识别带内环面要素。
