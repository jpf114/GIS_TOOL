# 矢量数据处理 / 边界提取

## 功能说明

按要素提取面边界线，并保留原有属性字段。

## 输入与输出
- 输入：面矢量数据
- 输出：边界线图层

## 参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| 输入文件 | 文件路径 | 是 | 无 | 输入矢量文件 |
| 输出文件 | 文件路径 | 是 | 无 | 输出线图层 |
| 图层 | 字符串 | 否 | 空 | 指定输入图层 |

## 算法内容

当前实现只支持面和多面图层。程序会对每条要素的标准化几何调用 `Boundary()`，把外环和内环边界提取成线结果，然后复制原属性写入输出图层。

输出几何类型固定按多线图层创建，但具体写入的边界几何可能是线或多线。空几何或无法得到边界的要素会被跳过。

## 处理流程

1. 打开输入图层并检查几何类型是否为面或多面。
2. 创建输出线图层，并复制空间参考和属性字段。
3. 逐要素读取输入几何并做标准化。
4. 对空几何直接跳过。
5. 调用 `Boundary()` 提取边界线。
6. 如果边界结果为空，则跳过该要素。
7. 复制原属性字段并写出边界结果。

## 输出说明

- 输出是线图层，不再保留面几何。
- 原属性字段保留。
- 一个输入面通常对应一条线或一组线。
- 多面和带孔面会输出更复杂的边界线结构。

## 推荐输出文件名

- `边界提取_result.gpkg`

## 注意事项

- 只支持面图层。
- 该工具不会做面之间的公共边界融合或去重。
- 带孔多边形的内环也会体现在输出边界中。

## 结果解释

- 输出线表示原几何的边界轮廓。
- 若结果中出现多条线，通常是多部件面或带内环面造成的。
- `skipped_count` 表示空几何或无可提取边界的要素数量。

## 适用场景

- 从面成果派生边界线成果。
- 后续做边界长度统计、线符号化展示。
- 需要把面轮廓单独输出给下游系统时使用。
