# 矢量数据处理 / 栅格化

## 功能说明

把矢量图层按指定分辨率烧写为单波段栅格。

## 输入与输出

- 输入：矢量图层
- 输出：单波段 `GeoTIFF` 栅格

## 参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| 输入文件 | 文件路径 | 是 | 无 | 输入矢量数据 |
| 输出文件 | 文件路径 | 是 | 无 | 输出栅格路径 |
| 图层名 | 字符串 | 否 | 无 | 用于读取范围和空间参考的目标图层 |
| 分辨率 | 浮点数 | 否 | `1.0` | 输出像元大小 |
| 属性字段 | 字符串 | 否 | 无 | 指定时把该字段值写入像元；留空则写入常数 `1` |

## 算法内容

程序先从目标图层读取外包范围，再按分辨率计算输出栅格的行列数，创建一个 `Float32` 的单波段 `GeoTIFF`。随后调用 GDAL 栅格化过程，把矢量几何烧写到栅格中。若未指定属性字段，则所有落入几何范围内的像元写成 `1`；未覆盖区域填 `0`，并把 `0` 设为 `NoData`。

## 处理流程

1. 打开输入矢量数据并定位目标图层。
2. 读取图层范围。
3. 根据范围和分辨率计算输出栅格宽高。
4. 创建单波段 `Float32` 的 `GeoTIFF`。
5. 写入地理变换和投影。
6. 用 `0` 初始化整幅栅格，并把 `0` 设为 `NoData`。
7. 若指定属性字段，则按字段值烧写；否则按常数 `1` 烧写。
8. 调用 GDAL 栅格化过程生成结果。

## 结果解释

- 输出是单波段 `Float32` 栅格
- 被矢量覆盖的像元值来自属性字段或常数 `1`
- 未覆盖区域为 `0`，并被标记为 `NoData`
- 返回消息中的尺寸是实际生成的栅格大小

## 适用场景

- 将行政区、地块、道路等矢量数据转成栅格参与像元级分析
- 为后续栅格运算准备掩膜图层
- 把分类编号或属性值写成像元值
- 将矢量边界转成规则网格输出

## 输出说明

- 当前实现固定输出为 `GeoTIFF`
- 输出范围来自输入图层外包矩形
- 输出投影直接继承输入图层坐标参考

## 推荐输出文件名

- `rasterize_result.tif`

## 注意事项

- 分辨率必须大于 `0`
- 输出尺寸由外包范围和分辨率直接计算，范围很大或分辨率很细时会生成很大的栅格
- 当前文档中的“图层名”与实现存在偏差：实现会用该图层读取范围和投影，但底层栅格化调用未显式限制到该单一图层
