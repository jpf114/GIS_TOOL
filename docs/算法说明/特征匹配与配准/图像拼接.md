# 特征匹配与配准 / 图像拼接

## 功能说明

将多幅输入影像拼接成一张全景结果。

## 输入与输出

- 输入：多个影像路径，使用英文逗号分隔
- 输出：拼接结果栅格，建议 `.tif`

## 参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| 输入文件 | 文件路径 | 是 | 无 | 多个输入路径用逗号分隔，至少 2 个。 |
| 输出文件 | 文件路径 | 是 | 无 | 输出拼接结果。 |
| 拼接置信度 | 浮点数 | 否 | `0.5` | `Stitcher` 的全景置信度阈值。 |

## 算法内容

图像拼接用于把多幅相互重叠的输入影像合成为一张连续结果。当前实现基于 OpenCV `Stitcher`，采用 `SCANS` 模式处理多景输入，适合连续扫描、条带或相邻视角较小的影像集合。

程序在进入拼接前，先逐幅读取每个输入文件的第 1 波段，把单波段数据转成 8 位灰度，再扩展成三通道 BGR 图像，以满足 `Stitcher` 的输入要求。拼接完成后，再把结果转回灰度图，并输出为单波段浮点栅格。

## 处理流程

1. 解析输入文件列表，要求至少提供 2 幅影像。
2. 逐幅读取各输入影像的第 1 波段。
3. 将每幅影像转成 8 位灰度图，再转换成 BGR 三通道图像。
4. 创建 `SCANS` 模式的拼接器，并设置 `拼接置信度` 阈值。
5. 调用拼接器自动完成特征匹配、几何估计、拼接融合。
6. 如果拼接失败，返回对应失败原因。
7. 将拼接结果从 BGR 转回灰度图，再归一化为浮点单波段栅格。
8. 以第一幅输入影像为参考写出输出文件。

## 填写示例

### 多文件输入

```text
D:/data/a.tif,D:/data/b.tif,D:/data/c.tif
```

### 输出说明

- 输出结果为栅格文件。
- 建议优先使用 `.tif` 保存，便于后续在 CLI、GUI 或 GIS 软件中继续处理。

## 推荐输出文件名

```text
D:/data/a_matching_stitch.tif
```

## 结果解释

- 输出影像是多景输入拼成的一张连续结果图。
- 元数据中的 `input_count` 表示参与拼接的影像数量。
- `output_width` 和 `output_height` 表示拼接后全景图尺寸。

## 适用场景

- 相邻航片、扫描条带或重叠影像的快速拼接。
- 对单波段结果做浏览、检查或后续裁切前的预处理。
- 对场景连续性要求高，但暂时不追求复杂几何控制的内部处理流程。

## 注意事项

- Debug lightweight 模式下该功能会被禁用。
- 代码当前按灰度方式读入，再转 BGR 参与拼接。
- 输出最终写成单波段浮点栅格。
