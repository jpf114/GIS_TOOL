# 特征匹配与配准 / ECC 配准

## 功能说明

使用 ECC 优化进行影像精配准，适合两景影像内容较接近的场景。

## 输入与输出

- 输入：参考影像 + 待配准影像
- 输出：配准后栅格，建议 `.tif`

## 参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| 参考文件 | 文件路径 | 是 | 无 | 参考影像。 |
| 输入文件 | 文件路径 | 是 | 无 | 待配准影像。 |
| 输出文件 | 文件路径 | 是 | 无 | 输出结果。 |
| ECC 运动模型 | 枚举 | 否 | `affine` | `translation` `euclidean` `affine` `homography`。 |
| ECC 迭代次数 | 整数 | 否 | `200` | 最大迭代次数。 |
| ECC 收敛阈值 | 浮点数 | 否 | `1e-6` | 终止阈值。 |
| 重采样方式 | 枚举 | 否 | `bilinear` | `nearest` `bilinear` `cubic`。 |
| 波段号 | 整数 | 否 | `1` | 从 1 开始。 |

## 算法内容

ECC 配准使用增强相关系数优化思想，直接在参考影像和待配准影像的灰度内容之间迭代求解几何变换参数。当前实现先读取指定波段并转成 8 位灰度图，再根据选择的运动模型初始化平移、欧式、仿射或单应性变换矩阵，然后调用 OpenCV 的 `findTransformECC` 逐步优化，使两幅影像在灰度分布上的相关性尽量提高。

与基于离散特征点的配准不同，这个算法不先提取关键点，而是利用整幅影像的灰度一致性求解变换，因此更适合同一区域、辐射差异不大、几何偏移相对平滑的精配准场景。求得变换矩阵后，程序再把原始输入栅格按参考影像尺寸重采样并输出。

## 处理流程

1. 读取参考影像和输入影像的指定波段。
2. 将两幅数据转成 8 位灰度图，作为 ECC 优化输入。
3. 如果输入尺寸与参考尺寸不一致，先把输入灰度图和原始波段数据缩放到参考尺寸。
4. 根据 `ECC 运动模型` 初始化对应维度的变换矩阵。
5. 按 `ECC 迭代次数` 和 `ECC 收敛阈值` 构建迭代终止条件。
6. 调用 ECC 优化，迭代求解让两幅图相关性最高的变换参数。
7. 按求得的变换矩阵对原始输入波段做几何重采样。
8. 按参考影像的空间尺寸和地理参考信息写出结果。

## 输出说明

- 输出结果为栅格文件。
- 建议优先使用 `.tif` 保存，便于后续在 CLI、GUI 或 GIS 软件中继续处理。

## 结果解释

- 输出影像表示已经对齐到参考影像网格的输入影像。
- 元数据中的 `motion_type` 表示实际使用的运动模型。
- 元数据中的 `warp_matrix` 为求解得到的变换矩阵，可用于排查配准是否稳定。

## 适用场景

- 同一区域不同时相影像的精配准。
- 特征点不明显，但灰度结构整体相近的影像对齐。
- 需要在参考影像网格上继续做变化检测、差值分析或叠加分析的预处理场景。

## 推荐输出文件名

- `ECC_配准_result.tif`

## 注意事项

- Debug lightweight 模式下该功能会被禁用。
- 如果输入和参考尺寸不一致，代码会先把输入缩放到参考尺寸再做 ECC。
- 元数据会返回运动模型和 warp 矩阵。

## 枚举值说明

### ECC 运动模型

- `translation`：仅平移
- `euclidean`：平移 + 旋转
- `affine`：仿射
- `homography`：单应性

### 重采样方式

- `nearest`：最近邻
- `bilinear`：双线性
- `cubic`：三次卷积
