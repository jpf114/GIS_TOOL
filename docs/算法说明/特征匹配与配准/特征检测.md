# 特征匹配与配准 / 特征检测

## 功能说明

在输入影像指定波段上提取关键点，可选导出为 JSON 文件。

## 输入与输出

- 输入：栅格影像，建议 `.tif`
- 输出：
  - 可不写输出文件，此时仅在结果中返回统计信息
  - 如果写输出文件，建议使用 `.json`

## 参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| 输入文件 | 文件路径 | 是 | 无 | 输入影像路径。 |
| 输出文件 | 文件路径 | 否 | 无 | 可选，导出关键点 JSON。 |
| 方法 | 枚举 | 否 | `sift` | 可选 `sift` `orb` `akaze`。 |
| 最大特征点数 | 整数 | 否 | `5000` | 检测上限。 |
| 波段号 | 整数 | 否 | `1` | 从 1 开始。 |

## 算法内容

特征检测用于从单幅影像中提取具有辨识度的关键点，并计算后续匹配可用的局部描述信息。当前实现支持 SIFT、ORB、AKAZE 三种方式，读取指定波段后统一转成 8 位灰度图，再调用对应检测器的 `detectAndCompute` 一次性完成关键点检测与描述子计算。

如果指定输出文件，程序会把关键点的空间位置、尺度、方向和响应值写成 JSON，便于后续比对不同方法的检测效果，或者为匹配、配准调参提供依据。

## 处理流程

1. 读取输入影像的指定波段。
2. 将波段数据转换成 8 位灰度图。
3. 按 `方法` 创建对应特征检测器，并设置最大特征点数量。
4. 在整幅灰度图上执行关键点检测和描述子计算。
5. 统计关键点数量。
6. 如果指定输出文件，则把关键点列表写为 JSON。
7. 返回关键点数量和使用的方法名。

## 输出内容

- JSON 文件包含关键点坐标、尺度、角度、响应值。
- 结果元数据包含 `keypoint_count` 和 `method`。

## 枚举值说明

### 方法

- `sift`：稳定性高，适合纹理较丰富场景
- `orb`：速度快，适合轻量场景
- `akaze`：速度和鲁棒性折中

## 输出文件示例

```json
{
  "keypoints": [
    {
      "x": 125.4,
      "y": 88.2,
      "size": 4.1,
      "angle": 35.0,
      "response": 0.02
    }
  ],
  "count": 1
}
```

## 输出说明

- 输出结果为统计或检查结果文件。
- 可直接用于后续汇总、质检或二次分析。

## 结果解释

- `keypoint_count` 反映当前算法和参数下检测到的关键点数量。
- JSON 中的 `size`、`angle`、`response` 可用于判断关键点尺度、方向和显著性。
- 关键点数量过少通常意味着纹理不足、波段选择不合适，或者该方法不适合当前数据。

## 适用场景

- 为后续特征匹配、自动配准提供前置评估。
- 比较 SIFT、ORB、AKAZE 在同一影像上的检测效果。
- 检查某个波段是否具备足够的结构信息用于特征类算法。

## 推荐输出文件名

```text
D:/data/scene_matching_detect.json
```

## 注意事项

- 输入影像之间的分辨率、波段和覆盖范围差异过大时，结果可能不稳定。
- 建议先检查输入是否可正常读取并具有可比性。
