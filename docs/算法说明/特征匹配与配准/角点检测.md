# 特征匹配与配准 / 角点检测

## 功能说明

提取 Harris 或 Shi-Tomasi 角点，可选导出为 JSON。

## 输入与输出

- 输入：栅格影像
- 输出：
  - 可不写文件，仅看统计
  - 若导出，建议使用 `.json`

## 参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| 输入文件 | 文件路径 | 是 | 无 | 输入影像。 |
| 输出文件 | 文件路径 | 否 | 无 | 可选，导出角点 JSON。 |
| 角点方法 | 枚举 | 否 | `harris` | `harris` 或 `shi_tomasi`。 |
| 最大角点数 | 整数 | 否 | `5000` | 输出角点上限。 |
| 质量阈值 | 浮点数 | 否 | `0.01` | 角点筛选质量阈值。 |
| 最小间距 | 浮点数 | 否 | `10.0` | 角点之间的最小距离。 |
| 波段号 | 整数 | 否 | `1` | 从 1 开始。 |

## 算法内容

角点检测用于提取影像中灰度在两个方向上都变化明显的位置。当前实现支持 `shi_tomasi` 和 `harris` 两种方法，但两者的处理方式并不完全一样。

当选择 `shi_tomasi` 时，程序直接调用 `goodFeaturesToTrack` 完成角点提取；当选择 `harris` 时，程序先计算 Harris 响应图，再按质量阈值筛选候选点，执行局部极大值判断，并通过最小距离抑制去掉过于密集的点，最终保留空间分布更均匀的角点结果。

## 处理流程

1. 读取输入影像的指定波段，并转成 8 位灰度图。
2. 根据 `角点方法` 进入不同分支：
   - `shi_tomasi`：直接按最大角点数、质量阈值和最小间距提取角点；
   - `harris`：先计算响应图，再筛选高响应候选点。
3. 对 Harris 候选点执行 3x3 局部极大值检查。
4. 按响应值从高到低排序，并结合 `最小间距` 做空间抑制。
5. 保留不超过 `最大角点数` 的结果。
6. 若指定输出文件，则把角点坐标等信息写成 JSON。
7. 返回角点数量和方法名。

## 输出内容

- 元数据返回 `corner_count` 和方法名。

## 枚举值说明

### 角点方法

- `harris`：Harris 角点
- `shi_tomasi`：Shi-Tomasi 角点

## 输出文件示例

```json
{
  "keypoints": [
    {
      "x": 210.0,
      "y": 96.0,
      "size": 3.0,
      "angle": -1.0,
      "response": 0.0
    }
  ],
  "count": 1
}
```

## 输出说明

- 输出结果为统计或检查结果文件。
- 可直接用于后续汇总、质检或二次分析。

## 结果解释

- `corner_count` 表示最终保留下来的角点数量。
- 角点通常集中在边界拐角、纹理交汇或结构突变的位置。
- 如果角点过密或过少，通常可通过 `质量阈值`、`最小间距`、`最大角点数` 进一步调整。

## 适用场景

- 提取稳定几何点位，供匹配、跟踪或人工检查使用。
- 对结构化场景做角点分布分析。
- 在不需要完整特征描述子的场景下，快速获取候选控制点位置。

## 推荐输出文件名

```text
D:/data/scene_matching_corner.json
```

## 注意事项

- 输入影像之间的分辨率、波段和覆盖范围差异过大时，结果可能不稳定。
- 建议先检查输入是否可正常读取并具有可比性。
