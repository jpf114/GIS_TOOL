# 特征匹配与配准 / 变化检测

## 功能说明

对前后时相影像执行变化检测，输出变化图。

## 输入与输出

- 输入：前时相参考影像 + 后时相输入影像
- 输出：变化强度栅格，建议 `.tif`

## 参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| 参考文件 | 文件路径 | 是 | 无 | 前时相影像。 |
| 输入文件 | 文件路径 | 是 | 无 | 后时相影像。 |
| 输出文件 | 文件路径 | 是 | 无 | 输出变化图。 |
| 变化方法 | 枚举 | 否 | `differencing` | `differencing` `ratio` `pcd`。 |
| 阈值 | 浮点数 | 否 | `0.0` | 小于等于 0 时自动按 `mean + stddev` 估阈值。 |
| 波段号 | 整数 | 否 | `1` | 从 1 开始。 |

## 算法内容

变化检测用于比较前后两景影像同一波段的像元差异强度。当前实现要求两幅影像尺寸一致，随后根据所选方法生成一幅连续值的变化强度图，而不是直接输出二值变化掩膜。

其中，`differencing` 使用绝对差值衡量变化幅度；`ratio` 先计算前后时相比值，再取对数绝对值，适合关注相对变化；`pcd` 把前后两景视作双通道数据，先求均值，再计算偏离均值后的联合变化幅度。阈值只用于统计变化像素数量和比例，不参与最终输出图像写出。

## 处理流程

1. 读取前时相参考影像和后时相输入影像的指定波段。
2. 检查两幅影像尺寸是否一致，不一致则直接返回失败。
3. 根据 `变化方法` 计算变化强度图：
   - `differencing`：计算绝对差值；
   - `ratio`：计算对数比值的绝对值；
   - `pcd`：以前后两景相对均值的偏差幅度表示变化。
4. 如果 `阈值` 小于等于 0，则对变化强度图统计均值和标准差，并用 `mean + stddev` 作为自动阈值。
5. 按阈值生成临时二值图，仅用于统计变化像素数量和比例。
6. 将连续值变化强度图转换成浮点栅格并写出。
7. 返回变化像素数、总像素数、变化比例和实际阈值等统计信息。

## 输出说明

- 输出结果为栅格文件。
- 建议优先使用 `.tif` 保存，便于后续在 CLI、GUI 或 GIS 软件中继续处理。

## 结果解释

- 输出栅格中的像元值越大，表示前后时相差异越明显。
- `changed_pixels` 和 `change_ratio` 是基于阈值统计得到的汇总指标，便于快速判断变化范围。
- 当自动阈值偏高或偏低时，结果统计可能与人工感知不完全一致，建议结合影像分布再调整阈值重跑。

## 适用场景

- 灾害前后、施工前后、季节变化等时序对比分析。
- 在正式分类或目标提取之前，先快速定位变化区域。
- 需要输出连续值变化强度图供后续阈值分级或专题制图使用的场景。

## 推荐输出文件名

- `变化检测_result.tif`

## 注意事项

- 两幅影像尺寸必须一致。
- 输出文件是变化强度图，不是二值 mask。
- 元数据会返回变化像素数量、总像素数、变化比例和实际阈值。

## 枚举值说明

### 变化方法

- `differencing`：差值法
- `ratio`：比值法
- `pcd`：主成分差分思路
