# 特征匹配与配准 / 特征匹配

## 功能说明

比较参考影像与输入影像的特征点，输出匹配统计，可选导出匹配结果 JSON。

## 输入与输出

- 输入：两景影像
- 输出：
  - 可不写输出文件，仅查看匹配统计
  - 若导出文件，建议使用 `.json`

## 参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| 参考文件 | 文件路径 | 是 | 无 | 参考影像。 |
| 输入文件 | 文件路径 | 是 | 无 | 待匹配影像。 |
| 输出文件 | 文件路径 | 否 | 无 | 可选，导出匹配点 JSON。 |
| 方法 | 枚举 | 否 | `sift` | 特征提取方法：`sift` `orb` `akaze`。 |
| 匹配方法 | 枚举 | 否 | `bf` | `bf` 或 `flann`。 |
| 最大特征点数 | 整数 | 否 | `5000` | 特征点上限。 |
| 比率阈值 | 浮点数 | 否 | `0.75` | Lowe ratio test 阈值。 |
| 波段号 | 整数 | 否 | `1` | 从 1 开始。 |

## 算法内容

特征匹配用于在两景影像之间建立关键点对应关系。当前实现先分别在参考影像和输入影像上提取局部特征，再用 BF 或 FLANN 匹配器做两近邻匹配，最后通过 Lowe ratio test 剔除歧义较大的候选匹配。

输出重点不是生成配准结果，而是提供匹配数量、关键点数量和可选的匹配点 JSON 文件，便于后续做质量检查、配准前评估或外部分析。

## 处理流程

1. 读取参考影像和输入影像的指定波段。
2. 将两幅波段数据转换成 8 位灰度图。
3. 按 `方法` 分别检测两幅影像的关键点并计算描述子。
4. 如果任一影像未检测到足够特征，则返回失败。
5. 按 `匹配方法` 对两组描述子执行两近邻匹配。
6. 按 `比率阈值` 进行 Lowe ratio test，仅保留稳定匹配。
7. 若指定输出文件，则把匹配点索引、距离和坐标写成 JSON。
8. 返回匹配数量以及两幅影像各自的关键点数量统计。

## 输出内容

- 元数据包含 `match_count`、参考图关键点数量、输入图关键点数量。
- 当任一影像检测不到足够特征时会失败。

## 枚举值说明

### 方法

- `sift`：稳定性高
- `orb`：速度快
- `akaze`：折中方案

### 匹配方法

- `bf`：暴力匹配，直接逐个比对
- `flann`：近似近邻匹配，大数据量下更快

## 输出文件示例

```json
{
  "matches": [
    {
      "idx1": 0,
      "idx2": 3,
      "distance": 12.5,
      "pt1": [120.0, 55.0],
      "pt2": [118.0, 57.0]
    }
  ],
  "count": 1
}
```

## 输出说明

- 输出结果为统计或检查结果文件。
- 可直接用于后续汇总、质检或二次分析。

## 结果解释

- `match_count` 表示通过比率检验后的有效匹配数量，通常比初始候选匹配更能反映实际匹配质量。
- `ref_keypoints` 和 `src_keypoints` 可用于判断两幅影像纹理信息是否充分。
- 如果关键点数量多但有效匹配少，往往说明两景内容差异较大，或参数设置不合适。

## 适用场景

- 在正式配准前评估两景影像是否适合自动匹配。
- 输出匹配点结果供外部工具或人工质检使用。
- 比较不同特征算法和匹配方法在同一数据上的效果差异。

## 推荐输出文件名

```text
D:/data/scene_matching_match.json
```

## 注意事项

- 输入影像之间的分辨率、波段和覆盖范围差异过大时，结果可能不稳定。
- 建议先检查输入是否可正常读取并具有可比性。
