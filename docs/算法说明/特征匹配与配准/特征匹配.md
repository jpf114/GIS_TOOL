# 特征匹配与配准 / 特征匹配

## 功能说明

比较参考影像与输入影像的特征点，输出匹配统计，可选导出匹配结果 JSON。

## 输入与输出

- 输入：两景影像
- 输出：
  - 可不写输出文件，仅查看匹配统计
  - 若导出文件，建议使用 `.json`

## 参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| 参考文件 | 文件路径 | 是 | 无 | 参考影像。 |
| 输入文件 | 文件路径 | 是 | 无 | 待匹配影像。 |
| 输出文件 | 文件路径 | 否 | 无 | 可选，导出匹配点 JSON。 |
| 方法 | 枚举 | 否 | `sift` | 特征提取方法：`sift` `orb` `akaze`。 |
| 匹配方法 | 枚举 | 否 | `bf` | `bf` 或 `flann`。 |
| 最大特征点数 | 整数 | 否 | `5000` | 特征点上限。 |
| 比率阈值 | 浮点数 | 否 | `0.75` | Lowe ratio test 阈值。 |
| 波段号 | 整数 | 否 | `1` | 从 1 开始。 |

## 输出内容

- 元数据包含 `match_count`、参考图关键点数量、输入图关键点数量。
- 当任一影像检测不到足够特征时会失败。

## 枚举值说明

### 方法

- `sift`：稳定性高
- `orb`：速度快
- `akaze`：折中方案

### 匹配方法

- `bf`：暴力匹配，直接逐个比对
- `flann`：近似近邻匹配，大数据量下更快

## 输出文件示例

```json
{
  "matches": [
    {
      "idx1": 0,
      "idx2": 3,
      "distance": 12.5,
      "pt1": [120.0, 55.0],
      "pt2": [118.0, 57.0]
    }
  ],
  "count": 1
}
```
