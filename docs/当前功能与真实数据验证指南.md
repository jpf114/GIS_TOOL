# 当前功能与真实数据验证指南

## 1. 这份文档解决什么问题

这份文档只回答四件事：

- 当前项目到底有哪些功能
- 每个功能现在有哪些参数
- 真实验证需要什么数据
- 这些数据能不能自己下载

## 2. 当前功能范围总览

当前项目共有 6 类插件：

| 插件 | 作用 |
|------|------|
| `projection` | 投影转换、坐标转换、坐标系赋值、信息查看 |
| `cutting` | 影像裁切、镶嵌、分块、波段合并 |
| `matching` | 特征检测、匹配、配准、变化检测、拼接 |
| `processing` | 阈值、滤波、增强、边缘、模板、分割、波段运算 |
| `utility` | 金字塔、NoData、直方图、信息、伪彩色、NDVI |
| `vector` | 矢量信息、过滤、缓冲、裁切、栅格化、格式转换、叠加分析 |

补充验证结论（2026-04-24）：

- Windows 下 `gis-cli.exe` 已验证可直接读取中文路径输入
- `vector filter` 已用真实 GeoJSON 数据验证中文属性条件可正常过滤
- 验证中文属性过滤时应直接使用原始数据文件，不要继续使用早期测试阶段遗留的中间副本

## 3. 通用输入要求

### 3.1 栅格数据

多数栅格类插件优先适合以下格式：

- GeoTIFF `.tif`
- 单波段或多波段栅格
- 最好自带地理参考和坐标系

### 3.2 矢量数据

多数矢量类插件优先适合以下格式：

- Shapefile `.shp`
- GeoJSON `.geojson`
- GeoPackage `.gpkg`
- KML `.kml`

### 3.3 坐标系

涉及空间运算时，最好保证输入数据坐标系一致，或明确给出源坐标系与目标坐标系。

## 4. 各插件详细功能、参数与真实数据要求

## 4.1 projection 插件

子功能：

- `reproject`
- `info`
- `transform`
- `assign_srs`

参数：

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| `action` | Enum | 是 | 无 | 子功能 |
| `input` | FilePath | 是 | 无 | 输入影像文件 |
| `output` | FilePath | 否 | 空 | 输出影像文件 |
| `dst_srs` | CRS | 否 | 空 | 目标坐标系 |
| `src_srs` | CRS | 否 | 空 | 源坐标系，覆盖影像自带坐标系 |
| `resample` | Enum | 否 | `nearest` | 重采样方法 |
| `x` | Double | 否 | `0` | 坐标转换 X |
| `y` | Double | 否 | `0` | 坐标转换 Y |
| `srs` | CRS | 否 | 空 | 用于 `assign_srs` |

重采样枚举：

- `nearest`
- `bilinear`
- `cubic`
- `cubicspline`
- `lanczos`
- `average`
- `mode`

真实数据要求：

- `reproject / info / assign_srs`
  - 一张带或不带坐标系的 GeoTIFF
- `transform`
  - 不一定需要真实文件，但最好有已知坐标系场景来核对结果

是否可自行下载验证：

- 可以
- 推荐公开数据类型：
  - Sentinel-2 / Landsat GeoTIFF
  - SRTM / NASADEM 高程 GeoTIFF

## 4.2 cutting 插件

子功能：

- `clip`
- `mosaic`
- `split`
- `merge_bands`

参数：

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| `action` | Enum | 是 | 无 | 子功能 |
| `input` | FilePath | 是 | 无 | 输入影像，多个文件可逗号分隔 |
| `output` | FilePath | 是 | 无 | 输出文件 |
| `extent` | Extent | 否 | `0,0,0,0` | 矩形范围 |
| `cutline` | FilePath | 否 | 空 | 裁切矢量 |
| `tile_size` | Int | 否 | `1024` | 分块大小 |
| `overlap` | Int | 否 | `0` | 分块重叠像素 |
| `bands` | String | 否 | 空 | 波段文件列表 |
| `dst_srs` | CRS | 否 | 空 | 镶嵌输出坐标系 |
| `resample` | Enum | 否 | `nearest` | 重采样方法 |

重采样枚举：

- `nearest`
- `bilinear`
- `cubic`
- `cubicspline`
- `lanczos`
- `average`

真实数据要求：

- `clip`
  - 一张 GeoTIFF
  - 如果走矢量裁切，再准备一个面矢量
- `mosaic`
  - 两张及以上、范围相邻或重叠的 GeoTIFF
- `split`
  - 一张较大的 GeoTIFF
- `merge_bands`
  - 多张空间范围一致、分辨率一致的单波段影像

是否可自行下载验证：

- 可以
- 推荐公开数据类型：
  - Sentinel-2 单景或相邻景
  - Landsat 多波段数据
  - 任意公开 GeoTIFF + 行政区边界面数据

## 4.3 matching 插件

子功能：

- `detect`
- `match`
- `register`
- `change`
- `ecc_register`
- `corner`
- `stitch`

参数：

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| `action` | Enum | 是 | 无 | 子功能 |
| `input` | FilePath | 是 | 无 | 输入影像 |
| `reference` | FilePath | 否 | 空 | 参考影像 |
| `output` | FilePath | 否 | 空 | 输出文件 |
| `method` | Enum | 否 | `sift` | 特征方法 |
| `match_method` | Enum | 否 | `bf` | 匹配方法 |
| `max_points` | Int | 否 | `5000` | 最大特征点数 |
| `ratio_test` | Double | 否 | `0.75` | Lowe 比率阈值 |
| `transform` | Enum | 否 | `affine` | 配准模型 |
| `resample` | Enum | 否 | `bilinear` | 配准重采样 |
| `change_method` | Enum | 否 | `differencing` | 变化检测算法 |
| `threshold` | Double | 否 | `0.0` | 变化阈值 |
| `band` | Int | 否 | `1` | 波段序号 |
| `ecc_motion` | Enum | 否 | `affine` | ECC 运动模型 |
| `ecc_iterations` | Int | 否 | `200` | ECC 最大迭代次数 |
| `ecc_epsilon` | Double | 否 | `1e-6` | ECC 收敛阈值 |
| `corner_method` | Enum | 否 | `harris` | 角点方法 |
| `max_corners` | Int | 否 | `5000` | 最大角点数 |
| `quality_level` | Double | 否 | `0.01` | 角点质量阈值 |
| `min_distance` | Double | 否 | `10.0` | 角点最小间距 |
| `stitch_confidence` | Double | 否 | `0.5` | 拼接置信度 |

关键枚举：

- `method`: `sift / orb / akaze`
- `match_method`: `bf / flann`
- `transform`: `affine / projective / similarity / translation`
- `resample`: `nearest / bilinear / cubic`
- `change_method`: `differencing / ratio / pcd`
- `ecc_motion`: `translation / euclidean / affine / homography`
- `corner_method`: `harris / shi_tomasi`

真实数据要求：

- `detect / corner`
  - 一张纹理丰富的影像
- `match / register / ecc_register`
  - 两张同区域、有重叠、视角变化不大的影像
- `change`
  - 两张同区域、不同时相影像
- `stitch`
  - 两张以上同类、部分重叠的影像

是否可自行下载验证：

- 可以
- 推荐公开数据类型：
  - Sentinel-2 同一区域不同时相
  - Landsat 同一区域多时相
  - 无人机或公开正射影像切片

说明：

- 这个插件对真实数据质量最敏感
- 如果两景之间没有足够重叠、分辨率差太大、云层遮挡太重，结果可能不好

参数：

| `output` | FilePath | 否 | 空 | 输出文件 |
| `band` | Int | 否 | `1` | 波段序号 |

| `threshold_value` | Double | 否 | `128.0` | 阈值 |
| `max_value` | Double | 否 | `255.0` | 阈值上限 |
| `filter_type` | Enum | 否 | `gaussian` | 滤波类型 |
| `kernel_size` | Int | 否 | `5` | 核大小 |
| `sigma` | Double | 否 | `1.5` | 高斯 sigma |
| `enhance_type` | Enum | 否 | `equalize` | 增强类型 |
| `clip_limit` | Double | 否 | `2.0` | CLAHE 参数 |
| `gamma` | Double | 否 | `1.0` | Gamma 值 |
| `expression` | String | 否 | 空 | 波段表达式 |
| `edge_method` | Enum | 否 | `canny` | 边缘方法 |
| `low_threshold` | Double | 否 | `50.0` | Canny 低阈值 |
| `high_threshold` | Double | 否 | `150.0` | Canny 高阈值 |
| `sobel_dx` | Int | 否 | `1` | Sobel dx |
| `sobel_dy` | Int | 否 | `1` | Sobel dy |
| `min_area` | Double | 否 | `100.0` | 轮廓最小面积 |
| `template_file` | FilePath | 否 | 空 | 模板文件 |
| `match_method` | Enum | 否 | `ccoeff_normed` | 模板匹配算法 |
| `pan_file` | FilePath | 否 | 空 | 全色影像 |
| `pan_method` | Enum | 否 | `brovey` | 全色锐化方法 |
| `hough_type` | Enum | 否 | `lines` | 霍夫类型 |
| `hough_threshold` | Double | 否 | `50.0` | 霍夫阈值 |
| `min_line_length` | Double | 否 | `50.0` | 最小线长 |
| `max_line_gap` | Double | 否 | `10.0` | 最大线间隙 |
| `min_radius` | Int | 否 | `1` | 最小圆半径 |
| `max_radius` | Int | 否 | `0` | 最大圆半径 |
| `circle_param2` | Double | 否 | `30.0` | 霍夫圆阈值 |
| `marker_input` | FilePath | 否 | 空 | 分水岭种子 |
| `k` | Int | 否 | `5` | 聚类数 |
| `max_iter` | Int | 否 | `100` | KMeans 最大迭代次数 |
| `epsilon_kmeans` | Double | 否 | `0.001` | 收敛阈值 |

关键枚举：

- `method`: `binary / binary_inv / truncate / tozero / otsu / adaptive_gaussian / adaptive_mean`
- `filter_type`: `gaussian / median / bilateral / morph_open / morph_close / morph_dilate / morph_erode`
- `enhance_type`: `equalize / clahe / normalize / log / gamma`
- `edge_method`: `canny / sobel / laplacian / scharr`
- `match_method`: `sqdiff / sqdiff_normed / ccorr / ccorr_normed / ccoeff / ccoeff_normed`
- `pan_method`: `brovey / simple_mean / ihs`
- `hough_type`: `lines / circles`

真实数据要求：

- `threshold / filter / enhance / edge / contour / stats`
  - 一张普通单波段或多波段影像即可
- `band_math`
  - 至少两波段的多光谱影像
- `template_match`
  - 一张主影像 + 一个从主影像中裁出的模板
- `pansharpen`
  - 一张全色影像 + 一张多光谱影像
- `watershed`
  - 目标边界较明显的影像
- `kmeans`
  - 任意多波段影像，最好类别差异明显

是否可自行下载验证：

- 可以
- 推荐公开数据类型：
  - Landsat 8/9
  - Sentinel-2
  - 公开 DEM
  - 公开航空影像

说明：

- `pansharpen` 最适合用 Landsat 8/9 的全色波段与多光谱波段验证
- `band_math` 和 `NDVI` 最适合用多光谱数据验证

- `overviews / nodata / histogram / info / colormap`

  - Sentinel-2

- `GeoJSON`
- `ESRI Shapefile`
- `GPKG`

- `info / filter / buffer / clip / convert / dissolve`
  - 一份矢量文件即可

- 可以

  - Natural Earth 行政区/河流/道路
  - OpenStreetMap 导出数据
  - GADM / GeoBoundaries 行政区
  - 任意公开分类栅格或自己生成的阈值结果栅格

## 5. 推荐真实数据下载组合

如果你想自己做一轮完整验证，最省事的组合如下。

### 5.1 栅格验证包

建议准备：

- 2 景同区域 Sentinel-2 或 Landsat 影像
- 1 景多波段影像
- 1 景高程或普通单波段 GeoTIFF

可以覆盖：

- `projection`
- `cutting`
- `matching`
- `processing`
- `utility`

### 5.2 矢量验证包

建议准备：

- 1 个行政区面数据
- 1 个道路或河流线数据
- 1 个地块或建筑面数据

可以覆盖：

- `vector`
- `cutting` 中的 `cutline`

## 6. 哪些功能最适合先验证

如果你想先从成功率最高的功能开始，建议顺序如下：

1. `utility/info`
2. `projection/info`
3. `projection/reproject`
4. `cutting/clip`
5. `processing/threshold`
6. `vector/info`
7. `vector/convert`
8. `utility/ndvi`
9. `matching/detect`
10. `matching/register`

## 7. 当前最实用的结论

- 大部分功能已经有实现和自动化测试支撑
- 真实数据验证完全可以自行下载并完成
- 最适合自行下载验证的数据是公开遥感影像 + 公开矢量边界数据
- 如果目标是“确认全部功能真的可用”，仍建议按插件逐项做真实数据回归
