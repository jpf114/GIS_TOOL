# 构建与开发说明

## 1. 目标

本文档用于说明 `GIS Tool` 在本地开发环境中的推荐构建方式、常见问题和日常开发约定。

## 2. 环境要求

推荐环境：

- Windows 10/11
- Visual Studio C++ 工具链
- CMake
- vcpkg

建议设置环境变量：

```powershell
$env:VCPKG_ROOT="D:\Code\GitHubCode\vcpkg"
```

## 3. 首次配置

项目默认会优先尝试使用 `VCPKG_ROOT` 提供的 toolchain。

首次配置命令：

```powershell
cmake -S . -B build -DGIS_BUILD_GUI=OFF -DGIS_BUILD_TESTS=ON
```

如果需要图形界面：

```powershell
cmake -S . -B build -DGIS_BUILD_GUI=ON -DGIS_BUILD_TESTS=ON
```

## 4. 编译命令

```powershell
cmake --build build --config Debug
```

## 5. 测试命令

```powershell
ctest --test-dir build -C Debug --output-on-failure
```

## 6. 当前已知问题

### 6.1 依赖初始化耗时较长

首次使用 `vcpkg` manifest 模式时，配置阶段可能耗时较长，因为会自动解析并安装依赖。

### 6.2 旧构建目录可能缓存错误配置

如果之前使用未带 toolchain 的构建目录配置过项目，后续即使修复了 `CMakeLists.txt`，旧目录仍可能沿用错误缓存。

建议做法：

- 直接使用新的构建目录
- 或显式清理旧构建目录后重新配置

### 6.3 GUI 并非默认开启

`Qt` 相关依赖体积较大，因此默认构建关闭 `GUI`，仅在需要时显式开启。

## 7. 开发约定

### 7.1 文档统一中文

- 新增文档使用中文
- 说明类注释优先中文表达清楚

### 7.2 提交信息统一中文

建议提交信息示例：

- `修复插件管理器销毁逻辑`
- `清理测试中的绝对路径依赖`
- `补充中文构建文档`

### 7.3 优先级原则

当前开发优先级：

1. 构建与测试基线
2. 插件框架稳定性
3. 插件测试覆盖与结构拆分
4. GUI 体验优化
5. 发布与打包

## 8. 推荐日常流程

1. 配置项目
2. 编译 Debug
3. 运行相关测试
4. 修改代码
5. 再次运行测试验证
6. 使用中文提交信息提交
