# 后续工作计划

## 目标

在 `0.1.3` 稳定版本基础上，继续提升项目的可用性、可维护性和发布质量。

## 近期工作

### 1. GUI 持续打磨

- 补强复杂参数的分组与引导
- 优化错误提示、执行反馈和结果展示
- 继续完善参数交互和数据工作流细节

### 2. 真实数据回归

- 按插件补齐更系统的真实数据验证
- 优先覆盖 `matching`、`vector`、`processing` 中对数据质量更敏感的功能
- 固化常用验证数据集与验证步骤

#### 当前仍缺的正式数据

- `NDVI`
  - 需要一景真实多波段遥感影像
  - 需要明确 `red_band` 和 `nir_band`
  - 最好同时提供波段说明或产品来源
- `pansharpen`
  - 需要一幅真实全色影像 `PAN`
  - 需要一幅与其配套的真实多光谱影像 `MS`
  - 两者应为同区域、同批次或严格配套产品
  - 最好同时提供多光谱波段说明

#### 数据到位后的验证顺序

- 先做 `CLI` 直跑，确认底层算法输出正常
- 再做 `GUI` 自动执行，确认与 `CLI` 输出一致
- 最后检查结果是否符合正式业务含义，而不只看程序是否成功运行

### 3. CI 与发布流程

- 在现有最小 Windows CI 基础上继续验证依赖安装时间和稳定性
- 视情况补充缓存策略
- 整理发布包结构与发布说明

## 中期工作

### 1. 插件实现拆分

- 继续降低 `processing`、`vector`、`matching` 等大文件复杂度
- 把公共校验、读写和结果组装逻辑继续下沉

### 2. CLI / GUI 一致性

- 统一参数说明、默认值表现和错误消息
- 减少同一功能在 CLI 和 GUI 之间的使用体验差异

### 3. 稳定版本管理

- 以“完整构建 + 完整测试 + 文档同步”作为后续版本发布基线
- 每次稳定发布都保持明确版本号与变更记录

## 当前原则

- 先保证可复现构建、可复现验证，再扩展功能
- 先做小步、可验证的改动，再做大范围重构
- 文档保持精简，只保留当前仍有实际价值的内容
