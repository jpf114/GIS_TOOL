#### 当前需要补齐的真实数据回归重点

- 第一优先级
  - `matching`
    - 这组功能对真实影像纹理、噪声、分辨率差异最敏感
    - 仅靠合成数据不能证明真实场景稳定性
  - `classification.feature_stats`
    - 需要真实分类栅格、真实行政区或地块面数据、真实类别映射
    - 这是用户场景最强的一组功能之一
  - `processing.pansharpen`
    - 需要真实 PAN / MS 配套影像
  - `utility.ndvi`
    - 需要真实多波段遥感影像并确认红光、近红外波段
- 第二优先级
  - `cutting`
    - `merge_bands` 需要真实多景同尺寸或可配准数据验证
  - `projection.info / transform / assign_srs`
    - 需要真实坐标点和缺失坐标系样例补验证
- 第三优先级
  - `processing.filter / contour / hough / watershed / kmeans / band_math / stats`
  - `matching.stitch`

#### 建议的数据准备方式

- 栅格
  - 一景真实单波段影像，用于 `histogram / colormap / threshold / edge / contour`
  - 一景真实多波段影像，用于 `ndvi / band_math / pansharpen / feature_stats`
  - 一组同区域可拼接影像，用于 `mosaic / stitch`
- 矢量
  - 一份真实道路或地块数据，用于 `vector` 系列
  - 一份真实行政区或统计区数据，用于 `feature_stats`
- 配准/变化检测
  - 同区域、不同时相或不同源的真实影像对
  - 至少准备一组“容易成功”的样例和一组“存在偏移/分辨率差异”的样例

#### 当前仍缺的正式数据

- `NDVI`
  - 需要一景真实多波段遥感影像
  - 需要明确 `red_band` 和 `nir_band`
  - 最好同时提供波段说明或产品来源
- `pansharpen`
  - 需要一幅真实全色影像 `PAN`
  - 需要一幅与其配套的真实多光谱影像 `MS`
  - 两者应为同区域、同批次或严格配套产品
  - 最好同时提供多光谱波段说明

#### 数据到位后的验证顺序

- 先做 `CLI` 直跑，确认底层算法输出正常
- 再做 `GUI` 自动执行，确认与 `CLI` 输出一致
- 最后检查结果是否符合正式业务含义，而不只看程序是否成功运行
