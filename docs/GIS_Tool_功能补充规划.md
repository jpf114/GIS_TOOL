# GIS Tool 功能补充规划

> 基于 C++17 + GDAL + OpenCV + PROJ + Qt 的插件式传统 GIS 算法工具集。
> 本规划仅针对本项目当前算法能力进行补充，不涉及 ONNX 或深度学习。

---

## 一、当前算法现状

本项目当前已形成多主功能插件体系，均为传统图像处理与 GIS 算子：

| 插件组 | 已实现功能 |
|--------|------------|
| projection | 重投影、坐标系信息查看、坐标转换、赋予坐标系 |
| cutting | 裁切、镶嵌、分块、波段合并 |
| matching | 特征检测、特征匹配、影像配准、变化检测、ECC 配准、角点检测、拼接 |
| processing | 阈值分割、滤波、增强、波段运算、统计、边缘检测、轮廓提取、模板匹配、全色锐化、霍夫、分水岭、K-Means、Gabor、GLCM、Mean Shift |
| classification | 地物分类统计、SVM、随机森林、最大似然分类 |
| georef | DOS、辐射定标、控制点配准、余弦校正、Minnaert、C 校正、QUAC、RPC 正射校正 |
| terrain | 坡度、坡向、山体阴影、TPI、曲率、粗糙度、填洼、流向、汇流累积、河网提取、流域划分、剖面提取、视域、多点视域、填挖方、库容计算 |
| raster_manage / raster_inspect / raster_render / raster_math | 金字塔、NoData、直方图、信息查看、伪彩色、波段运算 |
| spindex | NDVI、NDWI、自定义指数及其他已扩展光谱指数 |
| vector | 信息查看、过滤、缓冲、裁切、栅格化、面矢量化、格式转换、并集、差集、融合，以及一批拓扑检查与空间分析能力 |

现有功能已覆盖较完整的基础处理链，目前真正仍未落地的明显新增方向主要集中在**点云处理**；其余模块的重点已经从“有没有”转向“是否继续做实、做齐、做稳”。

## 二、插件与算法补充

### 2.1 插件补充：georef（几何校正与辐射处理）

**目标**：补齐遥感数据预处理环节，支撑遥感正射校正与大气校正。

- **RPC 正射校正**：基于有理多项式系数的正射纠偏，支持 DEM 辅助。
- **图像配准增强**：在现有特征匹配基础上，增加基于控制点文件的配准模式。
- **大气校正（简化版）**：暗像元法（DOS）、QUAC 快速大气校正，输出地表反射率。
- **辐射定标**：从 Landsat/Sentinel/高分系列元数据自动提取定标系数并转换。
- **地形校正**：C 校正、Minnaert 校正、余弦校正，减少地形阴影影响。

当前实现状态：

- 已完成：RPC 正射校正、控制点配准、DOS 大气校正、QUAC 大气校正、辐射定标（支持元数据文件自动读取 gain/offset）、余弦校正、Minnaert 校正、C 校正
- 待完成：无

**依赖**：GDAL（RPC 支持）、PROJ（坐标转换）、现有 feature matching 能力。

---

### 2.2 插件补充：terrain（地形分析）

**目标**：对 DEM/DSM 进行专业地形分析，支撑水文、视域等领域应用。

- **基础地形因子**：坡度、坡向、山体阴影、地形位置指数（TPI）、地表粗糙度。
- **水文分析**：填洼、D8 流向、汇流累积量、河网提取、流域自动划分。
- **视域分析**：单点/多点可视域计算，可设置观察点高度与目标高度。
- **体积计算**：填挖方分析、库容计算。
- **剖面提取**：沿路径提取高程剖面。

**依赖**：GDAL 栅格处理、现有 processing 模块的数学运算能力。

---

### 2.3 新增插件：pointcloud（点云处理）

**目标**：支持 LAS/LAZ 点云格式的基础处理与分析。

- **点云读写**：支持 LAS/LAZ 格式（通过 GDAL 点云驱动或自行集成 PDAL 读写）。
- **点云滤波**：统计离群点移除、体素降采样。
- **地面点分类**：基于布料模拟滤波（CSF）或简单形态学滤波。
- **点云栅格化**：生成 DSM/DTM，输出为 GeoTIFF。
- **高程归一化**：基于 DEM 将点云归一化到相对高度。

**依赖**：PDAL 库（可选）、GDAL 点云驱动。

**当前可行性结论**：

- 当前项目环境已通过 `tests/test_core.cpp` 中的 `GdalDriverCatalogProbe` 探针验证。
- 当前 GDAL 版本为 `3.12.3`，探针结果为 `pointcloud_drivers=<none>`。
- 当前 `D:\Develop\vcpkg\installed\x64-windows` 下也未发现可直接使用的 PDAL 头文件或运行库。
- 因此，`pointcloud` 当前不适合作为“直接开始实现”的下一主模块。
- 若后续继续推进，前置条件应是：
  - 先确认是否允许引入 PDAL；
  - 或先补齐具备 LAS/LAZ / COPC 能力的 GDAL 构建环境；
  - 在依赖明确前，只适合做方案调研，不适合承诺完整功能实现。

---

### 2.4 插件补充：spindex（光谱指数）

**目标**：在现有 NDVI 基础上，扩充常用遥感光谱指数库。

- **植被指数**：EVI、SAVI、ARVI、GNDVI、NDWI、NBR、BI（烧焦指数）。
- **水体指数**：MNDWI、AWEI。
- **城市指数**：NDBI、UI（城市指数）。
- **自定义指数构建器**：用户可自由组合波段计算表达式并保存为预设。

**依赖**：现有 processing 的波段运算能力。

---

### 2.5 现有 `vector` 插件增强

**目标**：从基础几何操作扩展到高级空间分析与拓扑处理。

- **拓扑检查与修复**：自相交检测与修复、悬挂节点检测、面重叠检测、碎片多边形消除。
- **高级空间关系分析**：最近邻分析、多边形邻接矩阵。
- **线简化与平滑**：Douglas-Peucker 简化、贝塞尔平滑。
- **空间统计**：核密度估计（点/线）、局部 Moran's I。
- **几何属性计算**：紧凑度、圆形度、走向等。

**依赖**：GEOS（已有）、GDAL/OGR 空间运算。

---

### 2.6 现有 `processing` 插件增强

**目标**：增加经典遥感分类与纹理分析算法。

- **监督分类器**：随机森林、SVM、最大似然分类（利用 OpenCV `ml` 模块）。
- **纹理分析**：GLCM 纹理特征（对比度、熵、同质性、相关性等）、Gabor 滤波。
- **图像分割**：MeanShift 分割、基于图的图像分割。
- **形态学分析**：开/闭运算系列、连通组件标记、骨架提取。

**依赖**：OpenCV `imgproc`、`ml`、`ximgproc`（如有）。

---

### 2.7 现有栅格工具体系增强

- **NDVI → 光谱指数迁移**：将 NDVI 功能升级为通用光谱指数计算器，纳入 `spindex` 插件。
- **影像统计增强**：支持按矢量区域分区统计（Zonal Statistics）。
- **直方图匹配**：一景影像匹配到另一景的色调分布。
- **COG 输出**：生成 Cloud Optimized GeoTIFF，便于云端发布。

当前实现状态补充：

- `utility` 已不再作为当前主插件存在
- 其中能力已拆分并归并到 `raster_manage / raster_inspect / raster_render / raster_math / spindex`
- GUI 当前进一步把 `raster_manage / raster_inspect / raster_render / raster_math` 合并展示为“栅格工具”主项
